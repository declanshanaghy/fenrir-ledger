B — Merge In Progress (Loading)

After user confirms, the join+merge operation may take 1–3s (multiple Firestore writes). Show a blocking progress state — no cancel allowed mid-merge.

Back button and Cancel are disabled/hidden during merge. Body scroll locked. If merge fails mid-way, show error state (Section D). Do NOT leave user with partial merge state visible.

D — Error States During Merge
MERGE FAILED — NETWORK ERROR
‹ Merge Failed
✕
Merge failed.
Your cards were not moved. Your solo household is intact. Please try again.

Idempotency is critical: if merge fails, the user's original solo household MUST remain intact. Engineer: implement merge as a Firestore transaction. API returns 500 on failure — client shows this error, user can retry or abandon.

RACE CONDITION — BECAME FULL
‹ Household Full
⚔
Household is now full.
Another member joined Eriksen Household while you were confirming. Your cards were not moved.

API returns 409 CONFLICT with reason=household_full even after initial validation succeeded (race condition). Client shows distinct message. No retry offered since household is full.

F — Accessibility Requirements (WCAG 2.1 AA)
  • Confirm button: aria-label="Merge 4 cards and join Eriksen Household" (full sentence for screen readers)
  • Warning block: role="alert" — read immediately by screen reader when state changes to error
  • Loading state: aria-live="polite" region announces "Merging your cards…" and "Merge complete"
  • Progress bar: role="progressbar" with aria-label="Merge progress" and aria-valuenow if determinate
  • Back button hidden during merge via aria-hidden="true" and tabindex="-1" (not just pointer-events: none)
  • Success screen: focus moves to "Go to Dashboard" button on appearance
  • Card list: <ul> with aria-label="Cards that will be merged"
