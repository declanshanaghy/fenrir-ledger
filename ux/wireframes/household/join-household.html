A — Desktop: Join Household Card (Idle / Empty State)

On desktop, the Join Household screen renders as a centered card within the app shell (same nav/header). Max width ~440px. URL: /ledger/join or /ledger/settings?join=1 (FiremanDecko to decide routing).

Join a Household
Enter the 6-character invite code from the household owner.
Invite Code

6 individual char inputs, auto-advancing. Accepts A–Z and 0–9 only (uppercase forced). Paste of 6-char string fills all boxes. Tab/Backspace navigate backward. See interaction-spec.md for paste & keyboard behavior.

Join button disabled until all 6 chars filled. No submit on partial code.

C — Edge Case: User Already In a Multi-Member Household

If a user who is already a member of a multi-member household navigates to Join, show a blocking state. They cannot join another household without leaving the current one (future scope).

D — Accessibility Requirements (WCAG 2.1 AA)
  • Each char box has aria-label="Character N of 6" and inputmode="text"
  • Fieldset wraps the 6 inputs with legend="Invite code (6 characters)"
  • Validation messages use role="alert" so screen readers announce them immediately
  • Loading state button has aria-busy="true" and aria-label="Checking invite code…"
  • Focus management: on error, focus returns to first char box; on success, focus moves to Continue button
  • All tap targets minimum 44×44px (char boxes: 44×56px on mobile)
  • Keyboard: Tab navigates between char boxes; Backspace from empty box moves focus backward
