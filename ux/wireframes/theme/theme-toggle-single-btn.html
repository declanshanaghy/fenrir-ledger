F. DOM Structure — Before / After

Before (remove)

<div
  role="radiogroup"
  aria-label="Theme"
  class="inline-flex items-center rounded-sm border border-border overflow-hidden"
>
  <button role="radio" aria-checked={isActive} aria-label="Light"
    class="... border-r border-border ..."
    style={{ minWidth: 44, minHeight: 36 }}>
    <Sun className="h-4 w-4" />
  </button>
  <button role="radio" aria-checked={isActive} aria-label="Dark"
    class="... border-r border-border last:border-r-0 ..."
    style={{ minWidth: 44, minHeight: 36 }}>
    <Moon className="h-4 w-4" />
  </button>
</div>

After (add)

<button
  type="button"
  onClick={() => setTheme(cycleTheme(resolvedTheme ?? theme))}
  class="flex items-center justify-center text-muted-foreground hover:text-gold transition-colors"
  style={{ minWidth: 44, minHeight: 44 }}
  aria-label={isDark ? "Switch to light mode" : "Switch to dark mode"}
>
  {isDark ? <Sun className="h-4 w-4" /> : <Moon className="h-4 w-4" />}
</button>

The THEME_OPTIONS array and ARIA role="radiogroup" structure are fully removed. The component returns a single <button>. No wrapper div, no border classes, no role="radio".

G. Accessibility Spec

Attribute Dark mode active (isDark = true) Light mode active (isDark = false)
aria-label "Switch to light mode" "Switch to dark mode"
Icon shown Sun (☀) — indicates action Moon (☾) — indicates action
type "button" (prevents form submission)
role implicit "button" (no override needed)
Keyboard Enter / Space activates toggle (native button behavior)
Focus visible Browser default focus ring; no suppression
Touch target 44×44px minimum

The dynamic aria-label is critical for screen reader users — it must describe the action, not the current state. "Switch to light mode" communicates what clicking will do, consistent with WCAG 2.4.6 (Headings and Labels).

H. Component Variant Strategy

ThemeToggle.tsx currently has three variants: "inline", "icon", "dropdown-icon".

Variant Current behavior Change?
"inline" Two-button radiogroup with borders YES — replace with single toggle button (this issue)
"icon" Single toggle button WITH a border (border border-border) OPTIONAL — can remove border to match new style; engineer's call
"dropdown-icon" Display-only icon for menu row (no click) No change

The "inline" variant is the only required change. The engineer may simplify by merging "inline" and "icon" into a single no-border toggle button, but that is out of scope for #1927 unless it simplifies the implementation.

I. Acceptance Criteria Traceability

  • Single <button> replaces div[role=radiogroup] with two buttons — shown in sections A, F
  • Icon swaps: sun when dark mode active, moon when light mode active — shown in sections B, F
  • No border, border-r, or border-border classes on toggle or container — shown in sections A, F
  • Click toggles between light and dark themes — shown in sections B, F (cycleTheme call)
  • aria-label reflects action, not current state — shown in sections B, G
  • Minimum 44×44px touch target — shown in sections C, F
