Void Norse — Dark Theme Contrast Palette

Issue #1875 · Luna UX Design · 2026-03-23 · Dark theme border visibility and thickness

Design Intent

The Void Norse dark theme evokes a Norse war-room at midnight — a forge lit by gold embers, ink-black stone walls, the glow of a distant hearth. The void-black background (hsl(28,15%,7%)) is intentional and striking.

The problem this wireframe addresses: the current border token (--border: 22 8% 20%) is only 13 lightness points above the background, producing a contrast ratio of approximately 1.4:1 — essentially invisible. Card edges, tab separators, button outlines, and divider lines all dissolve into the void. This is corrected by lifting the border token to hsl(28, 22%, 40%) and specifying minimum thicknesses per element class.

Scope constraint: this wireframe specifies changes to the dark theme only. The light theme (Vellum Norse) is unchanged. FiremanDecko must not touch :root when implementing these changes.
Thickness note: visual weight in dark themes depends on both color contrast AND stroke width. A 1px border at 3:1 contrast reads weaker than a 2px border at the same contrast. Both must be corrected together to meet the acceptance criteria.

Border Thickness — Visual Reference

All sample lines below use #725c47 (proposed border color) on a white background for clarity.

1px — current (too thin in dark)
1.5px — tabs, buttons, inputs, dividers
2px — cards, panels, modal outlines
The 0.5px jump from 1px to 1.5px is meaningful in dark themes: sub-pixel rendering on high-DPI screens can cause 1px lines to disappear at certain zoom levels when contrast is already marginal. 1.5px guarantees the line occupies at least 2 physical pixels on all screens.

Border Thickness — CSS Implementation Spec

Thickness is applied via component classes and Tailwind utilities — it is not a CSS custom property. FiremanDecko must update each component directly.

Element class / component Current thickness Required thickness Notes
Card, CardContent, ValhallaCardTile 1px (border) 2px (dark only) Use .dark .card { border-width: 2px; } or Tailwind dark variant
Tabs separator (TabsList / TabsTrigger bottom line) 1px 1.5px (dark only) Typically border-b; override in dark: [border-bottom-width: 1.5px]
Secondary / ghost button outline 1px (border) 1.5px (dark only) Applies to variant="outline" and variant="secondary"
Divider / Separator component 1px 1.5px (dark only) Shadcn Separator — override height (horizontal) or width (vertical)
Input / Select border 1px 1.5px (dark only) Input already gets focus ring — resting border thickness matters more
Nav sidebar border-right 1px 1.5px (dark only) Typically applied to Sidebar root element
Table row borders 1px 1px (unchanged) Table internal lines are intentionally lighter — no change
Modal / Dialog border 1px or none 2px (dark only) Modal outline against void bg must be card-level (2px)
Implementation pattern: use CSS .dark class overrides or Tailwind's dark: variant. Do not apply thickness changes to light theme. Suggested global rule in globals.css (outside @layer base):

.dark .border, .dark [class*="border-"] { /* targeted overrides per element, not global */ }

Prefer targeted per-component overrides to a global override that may cause unintended side effects.
