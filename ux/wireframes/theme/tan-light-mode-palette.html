What's New — Issue #1874: Contrast and Thickness Fixes

Three changes from v1.0 (#1833):
  1. --border darkened: hsl(35, 28%, 74%) → hsl(33, 36%, 48%) — the old value was too close to the parchment background (L=74% vs bg L=89%), borders were nearly invisible. New value achieves 3:1 UI component contrast against background.
  2. --muted-foreground darkened: hsl(28, 28%, 35%) → hsl(27, 35%, 28%) — the old value failed 4.5:1 AA when used on the muted surface (only 3.5:1). New value achieves 5.2:1 on the muted surface and 6.8:1 on page background.
  3. Border thickness specs added: all UI element borders now have explicit minimum thickness requirements — card borders 2px, all other elements 1.5px.
All other tokens from #1833 are unchanged. Dark mode is untouched. Odin's Throne variable mappings for --rune-border and --text-rune are updated to match.

Design Intent

The Vellum Norse palette evokes aged Norse manuscript vellum — the Codex Regius, the Flateyjarbok, the Great Sagas written on scraped calfskin. Warm amber-ochre backgrounds recall parchment aged by centuries. The foreground uses warm brown-black (iron-gall ink) rather than cool blue-black, anchoring the palette in the Norse world.

This palette is intentionally unified — the same HSL values applied to both Ledger and Odin's Throne, with only the CSS variable names differing between the two systems.

Luna's design rule: every tone in the palette must share the same warm hue family (30°–42° on the color wheel). No cool greys, no blue-tinted neutrals. The palette should read as a single unified material — parchment — at every depth.

Border Thickness Specifications — New in #1874

All borders in the Vellum Norse theme use --border (now hsl(33,36%,48%)) with the following minimum widths. These ensure physical visibility at standard screen densities.

Element Min Thickness CSS Property Reason
Card outlines 2px border: 2px solid hsl(var(--border)) Cards are primary information containers — a 2px frame gives sufficient visual weight against the parchment background
Input fields 1.5px border: 1.5px solid hsl(var(--border)) Inputs are interactive — 1.5px makes the field boundary clearly legible without feeling overly heavy
Secondary buttons 1.5px border: 1.5px solid hsl(var(--border)) Ghost/outline buttons need visible edges against all parchment surfaces
Tab separators 1.5px border-bottom: 1.5px solid hsl(var(--border)) Tab underlines at 1px were nearly invisible — 1.5px makes the active indicator readable
Divider lines 1.5px border-top: 1.5px solid hsl(var(--border)) Section dividers must be discernible as structural breaks, not noise
Sidebar border-right 1.5px border-right: 1.5px solid hsl(var(--border)) The nav/content separation line must clearly delineate the two areas
Footer separator 1.5px border-top: 1.5px solid hsl(var(--border)) Footer must read as a footer, not a faint hint
Active nav indicator 3px border-left: 3px solid hsl(var(--primary)) Unchanged from #1833 — uses --primary, not --border. Intentionally bold.
Focus ring 2px outline: 2px solid hsl(var(--ring)) Unchanged from #1833. Uses --ring (gold), 2px offset.
FiremanDecko: there is no new CSS variable for thickness — these are literal pixel values in component styles. Do not use 1px for any visible border. The minimum is 1.5px for small elements and 2px for cards.
