8. Implementation Reference

File / ComponentNotes
src/app/(marketing)/chronicles/page.tsxIndex page. getAllChronicles() sorted newest-first. Static generation.
src/app/(marketing)/chronicles/[slug]/page.tsxArticle detail. Compact header (no hero). Prev/Next above content. MDXRemote rsc.
src/app/(marketing)/chronicles/chronicle.cssScoped prose styles for .chronicle-page. Uses CSS vars for theme.
src/lib/chronicles.tsgetChronicleBySlug(), getAllChronicles(), getAllChroniclesSlugs().
stripInlineHeader()Removes duplicate <header> from MDX body (Issue #469).
stripInlineFooter()Removes duplicate <footer> from MDX body, global flag (Issue #486).
dedentHtml()Strips leading whitespace to prevent markdown code-block interpretation (Issue #407).
Theme toggleRotary single-click in nav (via sidebar-removal wireframe). next-themes.
No dual-realm crossfadeChronicles are dev logs, not lore content. Pure token-based theming only.
WCAG 2.1 AA — Chronicles Pages Both themes: 4.5:1 minimum for prose body text. Source Serif 4 at 14px (sm: 16px) base.
Prev/Next links: title is the visible label; aria-label not required but session number in title provides context.
Rune glyphs: aria-hidden=“true” on all decorative runes.
Back link: font-mono text-[0.65rem] at minimum — verify 4.5:1 against bg-background.
Code blocks: monospace at 10px min — use 12px minimum for WCAG compliance.
Interactive targets (cards, prev/next links, back link): min 44×44px touch target.
