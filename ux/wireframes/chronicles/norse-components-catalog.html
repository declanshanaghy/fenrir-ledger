CSS Architecture for MDX Implementation

Engineer reference: how the existing CSS and new Norse CSS should be organized.

FileScopeContains
chronicle.css.chronicle-pageExisting layout: session header, TOC, timeline, entries, user-msg, work-card, code-block, stat-grid, footer, agent-report layer (basic). Keep as-is, unchanged by this work.
chronicle-norse.css (NEW).chronicle-pageNorse ceremonial layer: .decree, .agent-callback, .heckle-comeback, .heckle-mayo, .heckle-entrance, .heckle-explosion, @keyframes norse-tremble, @keyframes explosion-glow. Imported in chronicle [slug]/page.tsx alongside chronicle.css.
Both files must be scoped under .chronicle-page. The HTML reports (generate-agent-report.mjs) keep their own inline CSS — they are NOT affected by this extraction. The shared CSS is for MDX only. CSS custom properties in chronicle-norse.css should use var(--c-*) tokens from chronicle.css (which maps to global theme tokens), NOT the raw hex values from the HTML generator CSS vars (--void, --gold, etc.). This ensures light/dark theme compatibility.
