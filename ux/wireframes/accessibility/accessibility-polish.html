Reduced Motion Support — prefers-reduced-motion: reduce

When the user has opted into reduced motion in their OS settings, all CSS keyframe animations and Framer Motion transitions must be suppressed or significantly reduced. Content must still appear — just without movement or flashing.

Animations to Suppress Under Reduced Motion
Animation Name Where Used Reduced Motion Treatment
saga-enter Page load stagger on card grid animation: none; opacity: 1; transform: none (immediate show)
saga-shimmer (skeleton) Loading skeletons animation: none (static grey skeleton is fine)
muspel-pulse StatusRing urgent pulse (≤30 days) animation: none (ring still shows — just static)
wolf-rise KonamiHowl wolf silhouette animation: none; FENRIR AWAKENS band still shows for 3s
saga-shake Body shake on Konami trigger Skip entirely — no shake
raven-warn HowlPanel header shake on new urgent card animation: none — header still updates
gleipnir-shimmer html.gleipnir-complete on unlock animation: none — Valhalla entry still appears
Framer Motion (card grid) AnimatedCardGrid enter/exit useReducedMotion() → skip AnimatePresence transitions (immediate mount/unmount)
Ragnarök overlay transition Opacity fade 1.2s / 0.8s Reduce to instant (transition: none) — overlay still appears/disappears

Framer Motion Pattern — AnimatedCardGrid
import { useReducedMotion } from 'framer-motion';

function AnimatedCardGrid({ cards }) {
  const shouldReduce = useReducedMotion();

  return (
    <AnimatePresence initial={false}>
      {cards.map(card => (
        <motion.div
          key={card.id}
          initial={shouldReduce ? false : { opacity: 0, scale: 0.96, y: 8 }}
          animate={{ opacity: 1, scale: 1, y: 0 }}
          exit={shouldReduce ? {} : { opacity: 0, scale: 0.94, y: -4 }}
          transition={shouldReduce ? { duration: 0 } : { duration: 0.35 }}
        >
          <CardChain card={card} />
        </motion.div>
      ))}
    </AnimatePresence>
  );
}
