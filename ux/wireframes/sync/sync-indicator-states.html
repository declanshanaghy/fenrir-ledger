Sync Indicator — States Wireframe

Issue #1125 · Component: SyncIndicator.tsx · Luna · 2026-03-16

Wireframe: layout and structure only. No theme colors applied. Engineer reads alongside theme-system.md.

A — Dot State Anatomy (all 4 states)

The dot is 8×8px, centered in a 20×20px invisible button hitzone. Fixed bottom-right. Four cloud sync states replace the current binary syncing/idle.

State 1: syncing

syncing

Dot: solid fill. Ping ring: full-width animated pulse expanding outward (animate-ping). Ring opacity ≈ 40%.

Token: --egg-accent (gold) — same as legacy syncing state.

Reduced motion: No ping ring. Dot remains solid fill, no animation. Color change alone signals state.

State 2: synced

synced

Dot: solid fill, no animation. Slightly brighter/more saturated than resting iron. Signals "cloud backup complete."

Token: success/green variant from theme — engineer discretion or new --sync-success token.

Duration: Stays in "synced" state for N seconds after last sync completes, then transitions to idle/dim. Engineer: expose SYNCED_DISPLAY_MS constant (suggest 3000ms).

State 3: offline

offline

Dot: dimmed, desaturated. 40% opacity or muted token. No animation. Conveys "sync paused — no network."

Token: --egg-border (dim iron) at reduced opacity, or new --sync-offline token.

No pulse: Offline state must never pulse — pulse implies active background work.

State 4: error

error

Dot: solid fill, red/destructive token. No animation. Conveys "last sync failed."

Token: --destructive or --sync-error.

Persists: Error state stays until next successful sync or user dismisses the toast. Does not auto-clear.

Aria: When state changes to error, aria-label updates to "Cloud sync failed". Consider role="status" on a visually-hidden sibling for screen readers.

Idle / pre-Karl state (existing + Thrall)

State 0: idle (dim)

idle — localStorage sync only (Thrall / offline)

Preserved from current implementation: --egg-border dim iron at rest. No cloud state shown for Thrall users — dot is always idle unless a local fenrir:sync event fires (existing behavior preserved).

B — Tooltip Variants (hover/focus, above dot)

Tooltip floats above the dot, appears on hover. Uses monospace font. Same position as existing Gleipnir tooltip. Tooltip content changes per state.

Thrall / idle (existing)

The sound of a cat's footfall

Gleipnir Fragment 1 tooltip — unchanged. Always shown for Thrall and idle Karl.

Karl — syncing

Syncing to Yggdrasil…

Shows cloud sync activity copy. Gleipnir tooltip suppressed during active sync.

Karl — synced

Backed up · just now

Relative timestamp: "just now" → "2 min ago" → "5 min ago". Updates live or on hover.

Karl — offline

Offline — sync paused

No action on click. Click still triggers Gleipnir if not yet discovered (offline does not block easter egg).

Karl — error

Tooltip links user to Settings for error details. Does not deep-link (just navigation hint in copy).

Gleipnir Fragment 1 preservation: The easter egg tooltip and click handler remain active for ALL states. Cloud sync tooltips appear IN ADDITION to the existing Gleipnir tooltip — the easter egg copy is shown for Thrall/idle; cloud copy replaces it only during active Karl sync states. Click always fires trigger() (no-op if already discovered). Cloud state tooltips do not disable the easter egg.

C — First Sync Confirmation Toast

Non-blocking toast shown after the FIRST successful cloud sync. Appears at bottom-center (or bottom-right, matching existing toast placement). Auto-dismisses after 5 seconds. Not shown on subsequent syncs.

Toast: first sync success

✓

Your 12 cards have been backed up

Yggdrasil guards your ledger.

"12" is the actual card count from the synced payload. Copy: "Your N cards have been backed up / Yggdrasil guards your ledger."

Trigger: fenrir:first-sync-complete event or flag in useCloudSync hook. Set localStorage fenrir:first-sync-shown = "true" to prevent repeat.

D — Sync Failure Toast (non-blocking)

Non-blocking toast. Bottom-right or bottom-center. Does NOT block interaction. Error dot persists on the indicator.

Toast: sync failure

!

Sync failed

Your cards are safe locally. We'll retry shortly.

Dismiss clears the toast but NOT the error dot — dot stays red until next successful sync.

"View details in Settings" navigates to /ledger/settings (scroll to sync section). Optional — engineer can omit if routing is complex.

State source: Driven by useCloudSync().status. Engineer maps hook return values to these 4 visual states.

G — Reduced-Motion & Accessibility

Reduced motion

@media (prefers-reduced-motion: reduce)

  • Suppress animate-ping ring entirely
  • Suppress any CSS transition on dot color (instant state change)
  • Dot remains solid fill — color change alone signals syncing
  • All other states unchanged (color still distinguishes states)

Tailwind: motion-reduce:animate-none on ping ring span. Or @media guard in CSS.

Keyboard / screen reader

  • Button is focusable (already in existing impl)
  • aria-label updates per state (see matrix above)
  • State changes: visually-hidden live region announces cloud state changes (syncing→error, syncing→synced)
  • Toasts use role="alert" for immediate announcement
  • Tooltip is aria-hidden="true" (decorative duplicate of aria-label)

Suggest: <span className="sr-only" aria-live="polite" aria-atomic="true"> sibling to button, updated by hook.
