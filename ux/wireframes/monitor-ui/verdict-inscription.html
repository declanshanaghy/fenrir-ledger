Section 0 — Context: Issue #1019 — NorseVerdictInscription

What this is: When a monitor-UI agent message contains a final handoff or QA verdict header ("Handoff" or "QA Verdict" in the last assistant text block), the markdown content is rendered inside a NorseVerdictInscription tablet instead of the standard assistant text bubble. If NorseErrorTablet is a tombstone, this is a TRIUMPH ARCH.

Relationship to NorseErrorTablet: Shares the parchment aesthetic (dark background, gold rune borders, Elder Futhark bands, Cinzel Decorative headings) but scaled to triumph-arch grandeur: double rune bands, larger central glyph, full markdown rendering, agent-specific color accent and rune, mythological pledge seal.

Context in the stream: Inline within the message stream (not full-pane like NorseErrorTablet). Replaces the outer .ev-assistant-text div. Max-width 720px, full-width on mobile.

Files affected: development/monitor-ui/src/components/NorseVerdictInscription.tsx, development/monitor-ui/src/styles/index.css (add .nvi-* classes), and the assistant text renderer that currently produces .ev-assistant-text.

Section 5 — Carve-In Entrance Animation

The "carve-in" effect should feel like runes being inscribed into stone — not a simple fade. Staggered reveal: outer structure appears first, then content is "carved" in from top to bottom. Respects prefers-reduced-motion.

0ms
Shell border appears — .nvi-shell fades from opacity 0, border-color animates from transparent to gold. Duration: 200ms ease-out.
50ms
Top rune band carves in — .nvi-rune-band-top slides down from translateY(-8px) + opacity 0 to resting position. Duration: 300ms ease-out. Rune characters also animate letter-spacing from 0.5em → final value (creates "etching" feel).
200ms
Agent glyph blazes in — .nvi-glyph scales from 0.6 → 1.0 with opacity 0 → 1. Simultaneously: text-shadow pulses from 0 → full glow → settled glow (simulates burning into stone). Duration: 400ms cubic-bezier(0.34, 1.56, 0.64, 1) (slight overshoot for impact).
350ms
Agent name + arch title fade in — .nvi-agent-name and .nvi-arch-title fade from opacity 0, translateY(4px) → resting. Duration: 300ms ease-out.
500ms
Content body reveals — .nvi-body children stagger with 60ms delay between each block (h1, h2, ul, table, pre each appear sequentially). fadeSlideIn from translateY(6px). Duration: 250ms per block.
final
Seal stamps in — .nvi-seal appears last with a brief scale pulse (0.95 → 1.02 → 1.0). The pledge text types in character-by-character (CSS animation on content, or JS interval). Bottom rune band mirrors top band's animation.

prefers-reduced-motion: When @media (prefers-reduced-motion: reduce) is active, collapse all animations to a single 150ms opacity fade. No translate, no scale, no stagger.

Implementation approach: Use CSS custom properties for delays, controlled by animation-delay on each region. A single carve-in keyframe handles opacity+translate; a separate glyphBlaze keyframe handles the glyph scale+shadow. The seal pledge typewriter effect should be a CSS animation on a pseudo-element, or a lightweight JS interval — NOT a third-party library.

Section 8 — Accessibility (WCAG 2.1 AA)

ARIA Structure

role="article" on .nvi-shell — semantically a self-contained verdict document within the stream.

aria-label="[AgentName] verdict inscription — [verdictType]" e.g. "FiremanDecko handoff inscription".

All rune bands: aria-hidden="true" — decorative Unicode characters must not be read by screen readers.

Agent glyph: role="img" + aria-label="[RuneName] rune — [meaning]".

Norse links: aria-label="[term] (opens Wikipedia in new tab)", target="_blank", rel="noopener noreferrer".

Pledge blockquote: aria-label="[AgentName]'s pledge to Odin".

The markdown content region: aria-label="Verdict content".

Color Contrast + Motion

All agent color accents must meet 4.5:1 contrast against the dark parchment background (#07070d). Engineer to verify each accent token meets this ratio.

Norse links must not rely solely on color for indication — the dashed underline and ↗ suffix provide shape-based cues.

prefers-reduced-motion: collapse all carve-in stagger to a single 150ms opacity fade. No translate, scale, or letter-spacing animation.

Focus states: Norse links must have visible :focus-visible ring (2px solid, offset 2px). No native outline removal without replacement.

Code blocks and tables must have meaningful accessible labels if they convey structured data — use <caption> on tables when the heading context is ambiguous.

Section 9 — Stream Placement & Detection

Detection rule: Examine the last assistant-role text block in the message stream. If its raw markdown contains a level-1 or level-2 heading matching /^#{1,2}\s+(Handoff|QA Verdict)/m, render as NorseVerdictInscription instead of .ev-assistant-text.

Scope: Only the LAST assistant text block per session stream triggers this. Mid-session "Handoff Notes" sub-sections in tool outputs do NOT trigger (they're inside code blocks or non-final messages).

Agent detection: Read agent identity from the session metadata (session.agentName field in the parsed stream). If unavailable, fall back to detecting agent name in the verdict text itself (e.g., "FiremanDecko → Loki Handoff" → agent is "FiremanDecko"). If still undetected → generic variant (gold accent, ᛟ Othalan rune — "heritage/estate").

Verdict type: Distinguish "Handoff" vs "QA Verdict" from the detected heading text. Handoff subtitles read "[AgentName] → [Recipient] Handoff". QA Verdict subtitles read "QA Verdict — [Status]".

Section 10 — Component Props Contract

// NorseVerdictInscription — Props Contract
// File: development/monitor-ui/src/components/NorseVerdictInscription.tsx

type AgentKey =
  | 'fireman-decko'
  | 'loki'
  | 'luna'
  | 'heimdall'
  | 'freya'
  | 'generic';        // fallback if agent is unidentified

type VerdictType = 'handoff' | 'qa-verdict';

interface NorseVerdictInscriptionProps {
  /** Raw markdown string — the full last-assistant text block */
  markdown: string;
  /** Identified agent — from session metadata or text detection */
  agent: AgentKey;
  /** Verdict type — detected from H1/H2 heading text */
  verdictType: VerdictType;
  /** Optional: recipient agent name for handoff subheadings, e.g. "Loki" */
  recipient?: string;
  /** Optional: session ID for aria-label */
  sessionId?: string;
}

// AGENT_VERDICT_CONFIG shape (all values are i18n-ready static strings):
interface AgentConfig {
  rune: string;                 // Elder Futhark character, e.g. "ᚲ"
  runeName: string;             // e.g. "Kenaz"
  runeMeaning: string;          // e.g. "The Forge"
  agentLabel: string;           // e.g. "FiremanDecko"
  agentFuthark: string;         // e.g. "ᚠᛁᚱᛖᛗᚨᚾ · ᛞᛖᚲᚲᛟ"
  archTitle: string;            // e.g. "The Forge Has Spoken"
  roleLabel: string;            // e.g. "Principal Engineer"
  accentCSSVar: string;         // e.g. "--agent-accent-fireman"
  dividerRune: string;          // repeated rune name for divider, e.g. "ᚲ ᛖ ᚾ ᚨ ᛉ"
  sealFuthark: string;          // e.g. "ᚠᛁᚱᛖ · ᛗᚨᚾ · ᛞᛖᚲᚲᛟ"
  elderName: string;            // e.g. "FiremanDecko — Kenaz — ᚲ"
  pledge: string;               // full pledge text (may include HTML for Norse links)
  pledgeNorseTerms: NorseTerm[]; // {term, wikiSlug} pairs pre-linked in pledge
}

interface NorseTerm {
  term: string;       // e.g. "Hliðskjálf"
  wikiSlug: string;   // e.g. "Hlidskjalf" (Wikipedia article slug)
}

Note on Norse term linking: Only the curated pledgeNorseTerms list within each agent's config should be auto-linked. Do NOT run a global dictionary replacement on the agent's markdown content — that could incorrectly link mid-sentence words the agent wrote without Norse-link intent. The agent's pledge text is controlled/static, so those links are safe to pre-define.
