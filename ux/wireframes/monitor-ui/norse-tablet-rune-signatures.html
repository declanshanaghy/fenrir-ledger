1 — Scope Overview

1A — Wikipedia Links Norse terms in both tablets wrapped in gold anchor tags → Wikipedia. Affects subheadings + body copy in NorseErrorTablet and inscription text in NorseTablet.
1B — Agent Rune Signatures New seal block at the bottom of NorseTablet. Shows dispatching agent's name + title in Elder Futhark + their unique epic quote. Driven by agentKey.
1C — Epic Seal — Error Tablet Replace generic "So it is written" with a variant-aware epic inscription. No agent context — uses a timeless, dramatic Yggdrasil-themed seal.
1D — Constants Extension AGENT_RUNE_NAMES, AGENT_RUNE_TITLES, AGENT_QUOTES added to constants.ts. Rune transliterations accurate Elder Futhark.

2 — Wikipedia Link Pattern

All Norse term links share a single visual pattern: gold-colored anchor, underline, external arrow glyph (↗), opens in new tab. WCAG 2.1 AA: links must be distinguishable from surrounding text via underline (not color alone). aria-label must describe the destination (e.g. "Yggdrasil on Wikipedia, opens in new tab").

Wikipedia links in NorseErrorTablet are limited to key terms in subheadings only (Yggdrasil, Bifröst). Do not link terms in the dynamic message body — content is user/system-generated and cannot be safely auto-linked.

2C — NorseTablet (Agent Prompt): Wikipedia Links in Inscription

The inscription text is the raw agent system prompt — it cannot be pre-linked. Wikipedia links in NorseTablet are NOT applied to the inscription text. Instead, links belong only in any static copy we add (e.g., the header title "Valhalla" reference if added, or future static preamble text). See interaction spec for the decision rationale.

Design Decision: NorseTablet inscription = raw system prompt text. Auto-linking Norse terms in arbitrary text risks false positives and accessibility regressions. Wikipedia links apply only to known static strings in NorseErrorTablet subheadings.

4 — Agent Rune Signatures: NorseTablet

The NorseTablet is the agent system prompt ("All-Father's Decree") tablet in LogViewer. The dispatching agent's agentKey is available via activeJob.agentKey. The rune signature block replaces the old generic seal line at the bottom of norse-tablet-body.

7 — Responsive Behavior

Mobile ≤375px — NorseErrorTablet Rune border font-size drops to 10px. Glyph reduces from 32px → 24px. Heading 22px → 18px. Seal inscription wraps to 2 lines — acceptable. Wikipedia link ↗ arrow remains visible inline. No layout change needed — single column always.
Mobile ≤375px — NorseTablet rune signature block Agent rune name font-size: 22px → 16px (still readable). Letter-spacing: 6px → 3px. Title runes: 13px → 11px. Quote max-width: none (fills container). Human-readable label below quote always visible. No horizontal overflow.
Desktop ≥1024px No change from base layout — tablets are always full-width within the LogViewer pane. Rune signature block max-width: 480px, centered within the tablet body.
