Closed and error share the same wolf-voice copy — both mean the connection is severed. The icon color differentiates them visually (grey vs red).

Use role="alert" for closed/error states so screen readers announce the loss of connection without the user hovering.

Tooltip implementation approach

Option A (preferred): Custom tooltip component in React with onMouseEnter / onMouseLeave / onFocus / onBlur handlers. Render tooltip as sibling in a position: relative wrapper. Uses aria-describedby + role="tooltip".

Option B (fallback): Native title attribute. Simpler but no styling control and no wolf-voice font rendering. Do NOT use if Option A is feasible within scope.

Engineer flexibility: if a shared Tooltip primitive already exists in the codebase, use it. The design only requires the copy and positioning behaviour specified here.

F. Collapsed Sidebar — Icon-Only Mode

When the sidebar is collapsed, the WSS icon is currently hidden (conditional render in Sidebar.tsx: {!collapsed && (...)}). The issue does not specify changing this behaviour. Design decision: keep icon hidden in collapsed mode — the sidebar is too narrow to safely show a tooltip without overlap.

If a future issue wants the icon in collapsed mode, the tooltip should appear to the right of the icon (not below it) to avoid viewport clipping.

Mobile touch behaviour

  • Tap icon → tooltip appears for 3 seconds, then auto-dismisses
  • Tap again → dismisses immediately
  • Touch target: min 44×44px (wrap icon in a transparent touch target if SVG is 16×16)
  • No hover on mobile — do not rely on CSS :hover alone

H. Accessibility (WCAG 2.1 AA)

Requirements

  • Keyboard access: Icon wrapper must be focusable (tabIndex={0} or button element). Tooltip shows on focus, hides on blur.
  • role="tooltip" on tooltip element. Linked via aria-describedby on the icon wrapper.
  • aria-label on icon wrapper: technical + mythological hybrid (see matrix, Section D). Do NOT put wolf-voice copy as aria-label — screen readers should convey actual status first.
  • role="status" or role="alert" on the StatusBadge span (see Section D matrix). Status changes should be announced via aria-live="polite" (connected) or aria-live="assertive" (error/closed).
  • prefers-reduced-motion: Suppress any pulse animation on the icon in connecting state.
  • Touch target: Wrap icon in a button or span with min 44×44px touch target on mobile (use padding or min-width/min-height).
  • Tooltip escape: Tooltip must dismiss on Escape key press.
  • No color-only distinction: Icon shape + color together encode state. The SVG plug/signal-arc metaphor is retained — not purely a colour dot. Passes against colour-alone reliance.
