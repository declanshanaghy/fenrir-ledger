Theme Variants — Marketing Site

Wireframe: ux/wireframes/marketing-site/theme-variants.html
Updated: Issue #642 — reflects current implemented pages
Pages: Home · Features · Pricing · About · Free Trial
Dual-realm lore crossfade pattern documented in §6

Wireframe Rules (enforced) Layout CSS only: flex/grid, border, width/height, padding/margin, font-size, font-weight.
No colors, no backgrounds (except #f8f8f8 on annotation blocks), no custom fonts, no shadows, no border-radius.
Theme styling lives in ux/theme-system.md — not here.
Theme System Reference (not in wireframe CSS — for implementation notes only) Light realm (Ljósálfar): bg-background=cream, text-foreground=dark, text-primary=accent, border-border=subtle.
Dark realm (Svartálfar): bg-background=deep, text-foreground=light, text-primary=accent, border-border=muted.
Tokens: Tailwind CSS vars via next-themes. resolvedTheme drives crossfade opacity.
Theme toggle: rotary single-click in profile dropdown (see sidebar-removal wireframe). System default: prefers-color-scheme.

6. Dual-Realm Lore Crossfade Pattern

Pattern: Both light and dark variants always in DOM. Opacity toggles 0↔1 via resolvedTheme from useTheme(). No mount/unmount — pure CSS transition. aria-hidden toggled for accessibility. Applies to: DualRealmPortrait, RealmBadge, RealmLore (about page) + ThemedFeatureImage (features page).

WCAG 2.1 AA Both themes: 4.5:1 for normal text, 3:1 for large text. Verify text-muted-foreground on bg-card meets 4.5:1 in both realms.
Myth links: dotted underline required (not color-only indicator).

Realm badge: aria-live=“polite” so screen readers announce realm change on theme toggle.
min-height: 44px on all interactive elements (buttons, CTAs, cards).
