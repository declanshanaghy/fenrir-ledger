Option B — Cinzel Decorative (font-display) — NOT RECOMMENDED for nav

Section 4 — Font-Display Strategy (No Layout Shift)

Both Cinzel and Cinzel Decorative are already loaded via next/font/google in development/frontend/src/app/layout.tsx. Next.js uses font-display: swap by default, which ensures:

Phase Behavior User impact
Font loading (FOIT window ~100ms) Swap period begins immediately. Fallback serif renders. Nav links visible almost instantly in fallback font.
Font loaded Browser swaps to Cinzel. Layout reflows only if metrics differ. Minor text reflow possible but nav height is fixed (h-16 / 64px).
Subsequent page loads Font cached — no FOIT, no swap, no layout shift. Instant Cinzel render from cache.

No changes to font loading configuration needed. The font-display: swap behavior is built into next/font. The nav bar fixed height (64px desktop, 56px mobile) prevents vertical layout shift even if horizontal character widths differ slightly during swap. FiremanDecko: do NOT add manual @font-face declarations — use existing CSS vars (--font-cinzel / Tailwind class font-heading) only.
