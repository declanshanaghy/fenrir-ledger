Card visibility rules (Thrall):
First 5 cards (by creation date) are fully visible and editable
Cards 6-N are shown as locked tiles with blurred/dimmed content
If user has 2+ locked cards, card 6 shows individually, card 7+ shows "+N more cards"
Clicking any locked card opens KarlUpsellDialog
Sort order for visibility:
Cards are ordered by creation date (oldest first)
This means the user's earliest cards remain visible — they have the most history
Data integrity:
All 12 cards remain in localStorage
Locked cards still count toward fee totals in any summary views
If user subscribes, all 12 cards immediately appear
"Add Card" button behavior on Thrall:
If user already has 5 cards: button opens KarlUpsellDialog
If user has fewer than 5: button works normally

Reference: Trial localStorage Keys
Trial lifecycle keys:
fenrir:trial_start_date — ISO date string, set on first card add
fenrir:trial_converted — "true" if user subscribed during/after trial
fenrir:trial_conversion_date — ISO date string, set on subscription
fenrir:trial_expiry_shown — "true" after expiry modal dismissed
fenrir:trial_midpoint_shown — "true" after day-15 toast shown
fenrir:post_trial_banner_dismissed — "true" after post-trial banner dismissed
fenrir:howl_alert_count — number, incremented on each Howl alert (for value summary)
Metrics computation:
Cards tracked: count all cards in fenrir:cards localStorage
Fees monitored: sum annualFee across all cards
Potential savings: sum annualFee of cards with status "closed" during trial period
Alerts received: fenrir:howl_alert_count
