Component Spec: TrialExpiryModal
Component:
TrialExpiryModal — shown once when trial expires
Trigger:
On app load when: fenrir:trial_start_date + 30 days < now
AND fenrir:trial_expiry_shown !== "true"
AND fenrir:trial_converted !== "true"
Modal behavior:
role="dialog" + aria-modal="true"
Focus trapped inside modal
Escape key closes (same as "Continue with free plan")
X button closes (same as "Continue with free plan")
Backdrop click does NOT close (intentional — user should make a choice)
z-index: 210 (standard modal layer)
Actions:
"Subscribe for $3.99/month" -- triggers Stripe Checkout flow
"Continue with free plan" -- sets fenrir:trial_expiry_shown = "true", closes modal
Post-subscribe:
Set fenrir:trial_converted = "true"
Set fenrir:trial_conversion_date = new Date().toISOString()
Remove trial badge, show KARL badge
All features unlocked
Post-decline:
Set fenrir:trial_expiry_shown = "true"
Show post-trial upgrade banner (dismissible, one-time)
Apply Thrall feature gates
Preserve all card data in localStorage
Accessibility:
aria-labelledby on modal title
aria-describedby on safety message
"Continue with free plan" is a visible button, not a link — equal to subscribe CTA
Both buttons meet 44px minimum touch target
Screen reader: announce value metrics via aria-label on the grid
Responsive:
Desktop: centered modal, max-width 500px
Mobile: w-[92vw] max-h-[90vh], scrollable body, feature comparison hidden
