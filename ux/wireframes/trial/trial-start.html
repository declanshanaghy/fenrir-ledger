Component Spec: TrialStartToast
Component:
TrialStartToast (uses existing sonner toast infrastructure)
Props:
cardCount: number — how many cards triggered the trial (1 for manual, N for import)
source: "manual" | "import" — determines body copy variant
Trigger conditions:
1. User saves first card via CardForm OR import wizard completes
2. fenrir:trial_start_date is NOT already set in localStorage
3. User is NOT already a Karl subscriber
Side effects on trigger:
1. Set fenrir:trial_start_date = new Date().toISOString() in localStorage
2. Show toast (8s auto-dismiss, sonner "success" variant)
3. After toast dismisses, trial badge appears in TopBar
Idempotency:
If fenrir:trial_start_date already exists, skip — trial already started.
If user is Karl subscriber, skip — no trial needed.
Accessibility:
Toast announced via aria-live="polite" (sonner default)
Dismiss button: aria-label="Dismiss trial notification"
Not focus-trapping — toast is informational, not blocking
Responsive:
Desktop (>1024px): bottom-right, max-width 380px
Tablet (600-1024px): bottom-right, max-width 340px
Mobile (<600px): bottom-center, full width minus 32px (16px margins)
Component Spec: TrialBadge (TopBar integration)
Component:
TrialBadge — inline badge rendered in TopBar, left of avatar
Render conditions:
1. fenrir:trial_start_date exists in localStorage
2. User is NOT a Karl subscriber (fenrir:trial_converted !== "true")
3. Trial has not been expired for more than 7 days (grace display period)
Props (computed):
daysRemaining: number — computed from trial_start_date + 30 days vs. now
Display text:
Days 30-26: "N days left" (neutral)
Days 25-2: "N days left" (neutral)
Days 5-2: "N days left" (warning — amber-hati theme color)
Day 1: "1 day left" (urgent — fire-muspel theme color)
Day 0: "Trial ends today" (urgent — fire-muspel theme color)
Expired: "Trial ended" (critical — red-ragnarok theme color)
Click behavior:
Opens TrialStatusPanel (see trial-status.html)
Accessibility:
role="status" on the badge container
aria-live="polite" — updates when day count changes
aria-label includes full context: "Karl trial: N days remaining. Click for details."
Minimum touch target: 44x44px (padding around text)
Mobile (375px):
Badge text shortens: "N days" (drops "left")
Font-size: 10px
Still clickable, still meets 44x44 touch target via padding
Karl subscriber:
Badge is NOT rendered. Replaced by "KARL" badge if desired (existing pattern).
