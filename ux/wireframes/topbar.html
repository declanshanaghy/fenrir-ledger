Scenario 1 (DEFAULT) — Anonymous · Desktop · Upsell prompt closed

Anonymous state (default for all users on first load).
The ᛟ rune (Othala) is the canonical anonymous identity. No generic user icon, no question mark, no silhouette.
No email is shown — there is none. No dropdown caret — this is not a user menu.
The dashed avatar border signals "unnamed / anonymous" in the wireframe. In implementation, the border uses the standard border-border token (not the gold ring, which is reserved for the signed-in state).
Clicking the avatar opens the upsell prompt panel (Scenario 2), not a profile dropdown.
Atmospheric copy for this state: "The wolf runs unnamed." — surfaced inside the upsell panel and nowhere else in the header.

Scenario 2 — Anonymous · Desktop · Upsell prompt open (via avatar click)

The wolf runs unnamed. Your chains are stored here alone.

Sign in to back up your cards and access them from any device. Your data is already here — signing in just keeps it safe.

The upsell prompt panel is a role="dialog" (not a menu).
Atmospheric copy ("The wolf runs unnamed.") uses Voice 2 — Norse saga.
The functional description below it uses Voice 1 — plain English.
Both CTAs are Voice 1: "Sign in to Google" and "Not now".
"Not now" closes this panel only. The dismissible upsell banner on the dashboard is a separate surface with its own dismiss.
Clicking "Sign in to Google" navigates to /sign-in — dedicated page, not an embedded OAuth redirect.
Keyboard: Escape closes the panel; focus returns to the avatar trigger.
Panel width: 260px desktop; calc(100vw - 32px) mobile.

Scenario 7 — Avatar Transition: Anonymous ᛟ rune → Signed-In Google avatar
1
Pre-transition (anonymous state): ᛟ rune in avatar circle. Dashed / neutral border. No email. No caret.
2
OAuth completes: The /sign-in page receives the Google token. Navigation returns to dashboard. If anonymous data exists, the migration prompt fires before the avatar updates (see migration-prompt.html).
3
Transition animation (implementation guidance): The avatar circle cross-fades from the ᛟ rune to the Google profile photo. The border transitions from the neutral border token to the gold ring (border-gold/40). Duration: ~400ms. Easing: cubic-bezier(0.16, 1, 0.3, 1) (same saga-enter curve). Simultaneously: email text fades in beside the avatar on desktop. Caret ▾ appears.
4
Post-transition (signed-in state): Google photo in avatar circle. Gold ring border. Email text (desktop). Caret ▾.

The transition is the "naming of the wolf" moment — the most emotionally significant state change in the product.
Non-negotiable: the Google photo must not just snap in — it must cross-fade.
The gold ring border appearing is the visual signal that the user is now named and backed up.
If the migration prompt fires (anonymous data found), the transition happens AFTER the user makes their choice in the migration dialog.
