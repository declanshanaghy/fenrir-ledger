Interaction Specification — Profile Dropdown

Trigger

Click or tap on the profile trigger button (avatar + email row on desktop; avatar only on mobile). Keyboard: Enter or Space while the trigger is focused.

Open behavior

  • Dropdown appears below the topbar, right-anchored.
  • aria-expanded on trigger changes to true.
  • Focus moves to the Sign Out button (first and only menu item).
  • An invisible backdrop covers the page behind the dropdown — click on backdrop closes.
  • Backdrop z-index sits between header (100) and the dropdown — do not use a full-screen opaque backdrop; the user should see the page behind.

Close behavior

  • Escape key: closes dropdown, focus returns to trigger.
  • Click / tap outside the dropdown: closes dropdown.
  • Tab past the last menu item: closes dropdown (no focus trap — user moves on).
  • Clicking Sign Out: signs the user out — dropdown closes as part of navigation away.

Sign Out action

  • Calls signOut({ callbackUrl: "/api/auth/signin" }).
  • No confirmation dialog — the action is immediate and intentional.
  • Session cookie is cleared server-side. Redirect to sign-in page.
  • Navigating back to "/" while signed out redirects to sign-in (protected route).

Avatar with picture <img alt="{name}'s profile photo" />

Profile dropdown role="menu", aria-label="User menu", id bound to trigger's aria-controls.
Sign Out button role="menuitem". Min 44×44px touch target.
Profile info block in dropdown aria-hidden="true" — name and email are surfaced via trigger aria-label on mobile.
Keyboard Escape closes dropdown, focus returns to trigger. Tab moves focus out of menu (closes). Enter/Space on trigger toggles.
