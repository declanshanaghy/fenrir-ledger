Timer Icon on Card Views — Issue #1850

Wireframe · Luna UX · 2026-03-23 · Covers: Hunt, Howl, Valhalla, All tabs · Mobile-first 375px+

Scope: Adds a second progress indicator (the Timer Ring) beside the existing spend/status ring on every card view. The timer fills from left to right (clockwise from 12 o'clock) based on elapsed time from openDate to the relevant deadline. The existing spend ring is unchanged.

1 — Icon Anatomy: Spend Ring + Timer Ring

Two 48×48px SVG rings sit side-by-side in the card header icon area. The left ring is the existing StatusRing (spend or fee progress). The right ring is the new TimerRing (time elapsed vs total window).

Existing: Spend Ring (StatusRing)

Spend ring
Issuer initial inside

No change. Existing StatusRing component. For Hunt: filled by amountSpent / spendRequired. For other tabs: filled by daysRemaining / totalDays. Color: realm-alfheim (Hunt bonus_open), realm-hati/muspel (fee urgency), etc.

New: Timer Ring (TimerRing)

Timer ring
Clock face inside

New component: TimerRing. Arc fills clockwise from 12 o'clock. Fill ratio = (today − openDate) / (deadlineDate − openDate), clamped [0, 1]. For Valhalla: ratio = (closedAt − openDate) / (deadlineDate − openDate), fully filled at 1. Inner icon: simplified clock face (circle + hour/minute hands as SVG lines) — no text inside.
Color mapping by tab: Hunt = realm-alfheim (teal), Howl = urgency thresholds (asgard/hati/muspel), Valhalla = realm-stone (muted).

Preferred Layout: Side-by-Side (4px gap)

Spend

Timer

Decision: side-by-side. Two 48×48 rings, 4px gap. Total icon footprint: ~100px wide × 48px tall. At 375px mobile this leaves ~275px for card name + status badge — sufficient. Alternative stacked (timer above spend, each 32px) is noted below but not preferred — reduces ring legibility.

Timer Ring Tooltip Spec

Hunt / active: "X days elapsed · Y days remaining until [bonus deadline]"
Howl (urgent): "X days elapsed · Y days until annual fee due"
Valhalla: "Held for X days (opened [date] → closed [date])"
Trigger: Native HTML title="" on the wrapper div (matches existing StatusRing pattern)
ARIA: role="img" aria-label="Timer: X of Y days elapsed toward [deadline]"

Alternative: Stacked Rings (not preferred — annotated for engineering reference)

Stacked: spend (left) + timer (right) both at 32px. Saves horizontal space but rings become harder to read. Not preferred.

Implementation note: HuntCardTile currently passes amountSpent/spendRequired as the ring numerator/denominator (reusing StatusRing math). The new TimerRing is a separate component — do NOT repurpose StatusRing for the timer. Both rings should be composable independently.

Howl Card — Design Decisions

Icon Area Layout — Howl

Rings shown:Timer ring ONLY (no spend ring)

Position:Same left position as dual-ring area on Hunt cards — consistent header height

Timer Fill Logic — Howl

Numerator:today − openDate (or lastFeeDate if available)
Denominator:annualFeeDate − openDate (365d fallback)
Color thresholds:>60d → realm-asgard · ≤60d → realm-hati (amber) · ≤30d → realm-muspel (red)
Urgent pulse:ring--urgent CSS class applied when ≤30d (same as StatusRing)

Howl Card Data Rows

Annual fee:card.annualFee (formatted currency)
Due date:formatDate(annualFeeDate) + "(Xd)" countdown when ≤60d
Bonus closing*:Shown only when bonus window ≤60d AND amountSpent < spendRequired
Opened:formatDate(card.openDate)
Action row:"Renew or cancel?" / "Action needed" + Manage/Review button

* Bonus closing row: Only appears when the card has both an upcoming annual fee AND an unmet bonus deadline within the same <60d window. This surfaces the double-urgency scenario where the user needs to act on both fee renewal AND bonus spend simultaneously.

Urgency Color Reference (Howl Timer Ring)

🔵 >60drealm-asgard (teal) — standard

Wireframe note: Color is conveyed here via emoji label only. In implementation use existing realm CSS tokens — do not hardcode colors.

7 — Accessibility

TimerRing ARIA

role:role="img"
aria-label (active):"Timer: X of Y days elapsed toward [deadline description]"
aria-label (Valhalla):"Timer: held for X days — closed [date]"
Clock face:aria-hidden="true" on the inner clock SVG lines (decorative)
Wrapper title:Native HTML title="" for mouse/pointer tooltip (matches StatusRing pattern)

Color Independence

Urgency:Conveyed by both color (realm tokens) AND stroke weight variation AND text countdown in data rows
WCAG 2.1 AA:Information must not be conveyed by color alone — the countdown text (e.g., "(18d)") satisfies this

Reduced Motion

ring--urgent pulse:Suppressed via @media prefers-reduced-motion (same as existing StatusRing)
Timer ring:No animation — static arc; no additional motion considerations

8 — Component Spec Summary (FiremanDecko Handoff)

New Component: TimerRing

File:src/components/dashboard/TimerRing.tsx
Props: openDate: string (ISO) · deadlineDate: string (ISO) · referenceDate?: string (ISO, defaults today) · closedAt?: string (ISO, Valhalla only) · tab: "hunt" | "howl" | "valhalla" · cardName?: string
Fill ratio:(referenceDate − openDate) / (deadlineDate − openDate) clamped [0, 1]
Valhalla ratio:Math.min(1, (closedAt − openDate) / (deadlineDate − openDate))
SIZE:48 (matches StatusRing constant)
RADIUS:18 (matches StatusRing)
Color token map:tab=hunt → realm-alfheim · tab=howl → urgency thresholds · tab=valhalla → realm-stone
Inner icon:SVG clock face: circle r=6, hour hand (vertical line 12 o'clock), minute hand (horizontal right)
Urgent class:ring--urgent applied when tab=howl AND daysRemaining ≤ 30

Modified Components

HuntCardTile:Add TimerRing beside StatusRing in icon area. Pass openDate + bonusDeadline. Tab="hunt".
HowlCardTile (new):New component (fee_approaching / promo_expiring). Timer ring only. Show annual fee, due date, optional bonus warning row, action row.
ValhallaCardTile (new):New component (closed/graduated). Both rings at 50% opacity + realm-stone. Bonus earned/✓/✗ state. Time held, closed date.
CardTile (base):Not modified — Hunt/Howl/Valhalla each use their own tile. All tab delegates to per-status tile.
Dashboard / DashboardTabs:All tab maps each card to its home tab tile component (no new All-specific tile).

Related issues: Builds on #1792 (Hunt min spend progress) and #1808 (Valhalla + Howl contextual cards). TimerRing is self-contained and can be developed independently of HowlCardTile / ValhallaCardTile.
