Wireframe: Trash Tab — Issue #1127

Dashboard tab navigation + Trash view. Deleted cards land here; users can restore or expunge. Thrall: locked with upsell. Free Trial + Karl: full access with Karl bling.

Sections: (A) Tab bar anatomy · (B) Trash view with cards · (C) Empty state · (D) Expunge single confirmation · (E) Empty Trash confirmation · (F) Thrall upsell state · (G) Karl bling treatment · (H) Mobile 375px · (I) Acceptance criteria

A — Tab Bar: Three Tabs (The Howl · Active · Trash)

Trash tab added at extreme right after Active. Tab order matches left-to-right reading priority: urgent → normal → deleted.

A1 — Trash tab resting (not selected):

Tab rune: ᛞ (dagaz — dissolution, transformation, the end of a cycle). Chosen for its Norse resonance with "ending". Unlike the destructive ᚦ (thurisaz), dagaz implies a meaningful transition rather than violence.
Tab label: "Trash" — plain English, scannable. Not "Valhalla" (that's a different Karl feature), not "Graveyard" (too dramatic for a utility tab).
Tab position: After Active, extreme right — mirrors macOS Trash convention: always rightmost/bottommost in navigation.
Tab badge: count of cards with deletedAt set. Gold border at var(--karl-gold) when count > 0 (Karl/trial tiers only).

A2 — Trash tab active (selected, Karl/trial tier):

Active state styling: Same border-bottom: 3px solid var(--karl-gold) as The Howl active state. Tab icon (ᛞ) gets karl-gold-glow pulse when selected and count > 0 — matching Howl tab's animate-muspel-pulse pattern but with gold variant.
Keyboard nav: Arrow keys cycle through all three tabs. Home/End jump to first/last. Same handleTabKeyDown pattern as existing DashboardTabs.

A3 — Thrall tier: Trash tab visible, locked indicator:

Thrall behavior: Tab is visible and clickable. Clicking fires KarlUpsellDialog — same pattern as Howl tab for Thrall users.
Lock glyph: ᛜ (ingwaz — seed/potential, sealed). Used consistently in SealedRuneModal.tsx for locked Karl features. Positioned inline after label, aria-hidden.
No count badge: Count badge hidden for Thrall — don't show how many cards they're missing, just tease the feature exists.
Aria-label: "Trash — upgrade to Karl to access" for screen readers (no visual label needed).

Card display rules: Deleted cards show name, issuer, last-4, and deletion date only. Fee dates, amounts, status badges, urgency bar — all hidden. They are dead cards; only their identity + deletion date matter.
Deletion date format: "Mar 13, 2026 · 3 days ago" — both absolute (for precision) and relative (for quick scanning). Relative text uses same formatRelativeTime utility as other card dates.
Card sort order: Most recently deleted first (newest deletedAt at top).
Restore icon ᚢ (uruz): Strength/vitality returning — appropriate for resurrection metaphor.
Expunge icon ᛟ (othala): Heritage/ancestral ending — dignified but final. Dashed border distinguishes from solid Restore border.

C — Empty State: No Deleted Cards

Shown when no cards have deletedAt set. No "Empty Trash" button visible (nothing to empty).

Empty state icon: ᛞ (dagaz) repeated — same tab icon, centered and enlarged. No decorative imagery needed.

Triggered by clicking "Expunge" on a trash card. Uses shadcn Dialog. Backdrop click / ESC / Cancel all dismiss without action.

Focus management: Dialog opens with focus on "Cancel" (safer default — destructive action not focused). Tab cycles Cancel → Expunge Forever → (wraps).
CTA label: "Expunge Forever" — emphasises irreversibility. Norse register stays consistent.
autofocus on Cancel: UX best practice for destructive confirmation dialogs.
Keyboard: ESC closes dialog (no action taken). Enter on focused button activates it.

Triggered by "Empty Trash" button in the Trash view header. Confirms expunge of ALL deleted cards at once.

Same focus and keyboard behavior as Section D.

F — Thrall Tier: Upsell on Trash Tab Click

Thrall clicks Trash tab → KarlUpsellDialog opens (same component as Howl, Valhalla, Velocity upsells). Below: the tab panel content that is never actually seen (just what would be shown to signal tab existence).

KarlUpsellDialog props for Trash tab (Thrall trigger):

  • featureName="Trash"

Tab panel for Thrall: When Thrall clicks the tab, do NOT render the panel at all — just open KarlUpsellDialog. The tab stays "unselected" visually (no aria-selected="true"). Prevents confusion about "I'm in the Trash tab but can't see anything".

Reduced intensity for trash cards: Trash cards are "dead" — their bling is intentionally subdued vs active cards. This creates visual hierarchy: active cards at full Karl intensity, trash cards at 50-60% of that, clearly in a secondary state.
CSS implementation: Add .trash-card rules inside existing [data-tier="karl"] and [data-tier="trial"] blocks in karl-bling.css. No new CSS files needed.
Hover on trash cards: No hover y-lift (Framer Motion AnimatePresence). Trash cards are static — no whileHover. Just a slight border-color brightening on hover.

FiremanDecko Implementation Notes

DashboardTabs.tsx changes:

  • Add TabId = "howl" | "active" | "trash"
  • Add trashedCards from cards.filter(c => c.deletedAt) — but note: Dashboard currently passes only non-deleted cards. TrashTab needs access to deleted cards from localStorage directly.
  • New prop on DashboardTabs (or sibling context): trashedCards: Card[]
  • Trash tab click for Thrall: intercept onClick, call setTrashUpsellOpen(true), do NOT call setActiveTab("trash")
  • Tab refs: extend tabRefs to include trash key

New component: TrashView.tsx (in src/components/dashboard/)

  • Props: trashedCards: Card[], onRestore: (cardId: string) => void, onExpunge: (cardId: string) => void, onEmptyTrash: () => void
  • Internal state: expungeTarget: string | null, showEmptyConfirm: boolean
  • Renders: header, card list (sorted by deletedAt desc), empty state, two confirmation dialogs

Storage layer changes (storage.ts):

  • getDeletedCards(householdId) — returns cards where deletedAt is set
  • restoreCard(householdId, cardId) — clears deletedAt, returns updated card
  • expungeCard(householdId, cardId) — removes card record entirely from localStorage
  • expungeAllCards(householdId) — removes all records with deletedAt set

Karl sync on restore (Karl tier only):

  • After restoreCard(), if tier === "karl": trigger existing cloud sync mechanism
  • Same pattern as card create/update sync flow

Karl bling CSS (karl-bling.css):

  • Add [data-tier="karl"] .trash-tab .tab-icon animation rule
  • Add [data-tier="karl"] .trash-card border rule (0.30 opacity gold)
  • Add [data-tier="trial"] .trash-card border rule (0.20 opacity gold)
  • Rune corners on .trash-card: inherit from .karl-bling-card .rune-corner but reduce opacity
