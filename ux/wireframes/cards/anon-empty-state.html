Issue #1748 — Anon Cards Screen: Sign-in CTA + Secondary Add Card
The anonymous (logged-out) empty state at /ledger currently buries the sign-in link as a subtle muted text line above the hero. This wireframe defines the new hierarchy: sign-in as primary CTA, add-card-locally as a clearly secondary option. Interaction spec: anon-empty-state-interaction-spec.md.
(A) Current State — Problem Reference
What anonymous users see today. Sign-in is a tiny muted text link above the hero; "Add Card" is the visually dominant action. Hierarchy is inverted for a product that needs account creation as its primary funnel.

Problems
1. Sign-in link is tiny, muted, above the hero — not scannable as primary action.

4. No trial value proposition presented at point of decision.
(B) New Design — Anonymous Empty State — Desktop
Primary CTA: Start free trial / Sign in. Secondary CTA: Add a card locally. The value proposition is front-and-centre. The local-add path is available but visually subordinated. The SignInNudge subtle-text mode is replaced entirely by this in-place hierarchy — remove the SignInNudge when hasCards=false.

Design Decisions
1. Primary button uses full border-weight (2px) and bold type — largest interactive element on screen.
2. Secondary button uses 1px border, smaller font — clearly subordinate but still easily tappable (44px+ touch target).
3. "or" divider makes the binary choice explicit without a visual line/rule that adds noise.
4. Footnotes under each CTA provide context without cluttering the buttons.
5. The SignInNudge component's hasCards=false branch is deprecated — remove it to avoid duplication.
6. Atmospheric heading preserved: Gleipnir/Fenrir copy stays — it's brand voice, not wayfinding.

(F) Component Structure — Implementation Notes
Recommended approach for FiremanDecko. Two implementation paths are offered; Path 1 is preferred for minimal blast radius.
Path 1 — AnonEmptyState component (preferred)
New file: components/dashboard/AnonEmptyState.tsx
Renders: Heading + subtext + CTA group (Sign in primary, Add Card secondary)

Dashboard.tsx change:
Import useAuth. In the zero-cards early return:
if (isAnon) return <AnonEmptyState />;
return <EmptyState />; // authenticated

SignInNudge.tsx change:
Remove the hasCards=false branch (the <p> subtle-link block). The hasCards=true banner branch is unchanged.
Why preferred: EmptyState stays focused on the authenticated case. AnonEmptyState owns the full conversion-optimised anon flow without mixing concerns.
Path 2 — Extend EmptyState with isAnonymous prop
Modified: EmptyState.tsx — accept isAnonymous?: boolean prop
Conditional inside EmptyState:
When isAnonymous=true: render sign-in primary + add-card secondary
When isAnonymous=false (default): render current import + add-card

Dashboard.tsx change:
Pass isAnonymous from useAuth:
return <EmptyState isAnonymous={status !== "authenticated"} />;
Acceptable if keeping the component count low is preferred. More prop-threading required.
SignInNudge Deprecation Scope
Only the !hasCards branch (lines 66-78 of SignInNudge.tsx) is removed. The hasCards=true full banner (lines 81-188) is unchanged. The prop interface stays the same; the component just never renders the subtle-link mode.
