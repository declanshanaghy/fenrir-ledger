A — Tab Bar: The Howl Active State (Desktop)

The Howl tab selected. Badge count reflects total cards with upcoming fees or expiring promos. Badge border reflects highest urgency level present.

Tab badge urgency: When Howl has cards <30 days: badge border becomes var(--urgency-red). When all cards 30–60 days: badge border becomes var(--urgency-amber). When no Howl cards: badge shows 0 at reduced opacity (same as other tabs). This is the ONLY place color signals urgency in the tab bar — no animation on the tab itself at this breakpoint.

B — Urgency Color System

Two urgency tiers. Applied to: card row left-border, urgency badge, date/deadline field values, summary bar "nearest deadline" stat.

RED (critical): <30 days until annual fee or bonus deadline. Token: var(--urgency-red). Label: "X DAYS" or "DUE SOON" (≤7 days).
AMBER (warning): 30–60 days until annual fee or bonus deadline. Token: var(--urgency-amber). Label: "X DAYS".

Urgency is computed from the sooner of: annualFeeDate OR bonusDeadline. If a card has both, the earlier date drives the urgency level. Implementation: compute daysUntilSoonest(card) → urgencyLevel. Do not rely on color alone — the date/days value is always shown in text alongside the color cue.

Reduced motion: No pulsing animation on urgency indicators when prefers-reduced-motion: reduce. Static color only.

H — Accessibility & Implementation Notes for FiremanDecko
Accessibility (WCAG 2.1 AA)
  • Tab panel: role="tabpanel", aria-labelledby pointing to Howl tab button
  • Urgency color: NEVER used alone. Date/days text always present alongside color. Urgency badge includes "X days until action required" as aria-label
  • Card row left-border urgency: decorative (aria-hidden). Card's aria-label includes urgency info: "X days until [fee|deadline]"
  • Progress bar: role="progressbar", aria-valuenow=(spent), aria-valuemin=0, aria-valuemax=(goal), aria-label="Bonus spend progress for [card name]"
  • "Act now" / "View tips" buttons: aria-label includes card name
  • Snooze button: aria-label="Snooze [card name] alert". After snooze: aria-label updates to "Snooze [card name] alert (snoozed Nx)"
  • Summary bar: role="region", aria-label="The Howl summary — action required"
  • Empty state: aria-label="The Howl empty state". Rune aria-hidden="true"
  • Color-only urgency violations: urgency-red and urgency-amber MUST pass 4.5:1 contrast ratio on card background. Engineer to verify with theme tokens.
Component / Data Notes
  • Data fields needed per card: annualFeeAmount (number|null), annualFeeDate (Date|null), bonusDeadline (Date|null), bonusMinSpend (number|null), bonusSpentSoFar (number|null), actionText (string)
  • annualFeeAmount = 0 or null → render "$0 (no fee)" in Annual Fee field
  • bonusDeadline = null → render "— (no active bonus)" in Bonus Deadline field, hide progress bar
  • urgencyLevel per card: Math.min(daysUntil(annualFeeDate), daysUntil(bonusDeadline)) → <30 = red, 30–60 = amber
  • Cards with urgencyLevel > 60 days should NOT appear in Howl tab
  • Sort: ascending daysUntilSoonest. Cards re-sorted on each render (days change daily)
  • Summary bar: max urgency across all cards drives accent color. Nearest deadline = min(daysUntilSoonest)
  • Snooze: localStorage key per card "howl-snooze-[cardId]" = ISO timestamp. Auto-expires when snoozed date passes. Cannot snooze past annualFeeDate − 3 days
  • actionText: engineer may auto-generate or store per-card. Suggest auto-generation from fee/deadline data for MVP.
  • Relevant component: HowlTab.tsx within DashboardTabs. Shared SpendProgressBar component with Hunt tab.

Relationship to Howl Panel (chrome/howl-panel.html): The Howl Panel is a NOTIFICATION SIDEBAR triggered by urgent cards. This wireframe is the DASHBOARD TAB VIEW — full-width content area, consistent with other tabs. Both surfaces should show the same cards but with different layout density. The panel is a quick alert; the tab is the full management surface. FiremanDecko: confirm whether both share a data layer or if HowlTab is the canonical source.
