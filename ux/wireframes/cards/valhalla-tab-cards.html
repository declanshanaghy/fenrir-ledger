G — Accessibility & Implementation Notes for FiremanDecko
Accessibility (WCAG 2.1 AA)
  • Tab panel: role="tabpanel", aria-labelledby pointing to Valhalla tab button
  • Card list: role="list" on wrapper, role="listitem" on each article
  • Card details grid: role="list" / role="listitem" — each detail-item announces label+value pair to screen reader
  • Sign-up bonus "earned" color: must pass 4.5:1 contrast on card background. Do not rely on color alone — label "Sign-up Bonus" already disambiguates
  • Empty state: aria-label="Valhalla empty state". Rune aria-hidden="true"
  • Summary bar: role="region", aria-label="Valhalla summary — total rewards reaped"
  • "View record" buttons: aria-label includes card name to disambiguate duplicate labels
  • Sort/filter selects: aria-label on each select element
Component / Data Notes
  • Data fields needed per card: signupBonusEarned (number+type), annualFeePaid (number), timeHeld (months), closedDate (Date), rewardsType ("points"|"miles"|"cashback")
  • signupBonusEarned = null when bonus was not earned — render as "— (not earned)"
  • annualFeePaid = 0 renders as "$0 (no fee)"
  • Summary bar totals: sum all signupBonusEarned by type, sum all annualFeePaid, calculate avg timeHeld, compute netRewardValue (est. value − fees)
  • Summary bar dismiss: localStorage key "valhalla-summary-dismissed", re-shows when new card is closed
  • Card sort default: closedDate descending (newest first)
  • Relevant component: ValhallTab.tsx (or similar) within DashboardTabs
  • Left border color token: var(--stone-hel) — the muted/archive tone already used in app/valhalla.html tombstones

Scope: This wireframe covers the DASHBOARD TAB card row view for Valhalla. The standalone full-page Valhalla archive (app/valhalla.html) is separate — it may link here or be reached via "View record". The tombstone detail view is out of scope for #1808.
