Notes

Phase 1: one provider button. The layout is not sparse — a single full-width button with a clear label reads purposeful, not unfinished. No empty space below it; the divider and "Continue without signing in" fill the rhythm.

No × close button. The only dismiss path is "Continue without signing in." Rationale: a × in the corner creates a second, visually smaller exit that competes with the prominent exit below. One clear path is better than two ambiguous ones.

Escape key also closes the dialog (implementation detail — handled by the Radix Dialog primitive).

Trigger points: (a) ᛟ avatar tap in TopBar, (b) "Sign in to sync" in upsell banner, (c) Settings "Sync to cloud" option.
Dialog element: <dialog> with aria-modal="true" and aria-labelledby bound to H1.
Focus trap: on dialog open, focus moves to first interactive element (GitHub button). Tab cycles within dialog only.
Backdrop: z-index: 200; dialog: z-index: 210. Clicking backdrop does NOT close (anonymous-first: no accidental dismiss).

Notes

4 providers fit cleanly in the vertical stack without scrolling at 360px dialog width. The dialog grows taller — max content height stays well within viewport bounds on both desktop and mobile (measured at 375px: ~480px total dialog height, leaving room above the fold on a 667px iPhone SE).

"Continue without signing in" remains equally visible with 4 providers as with 1. The divider creates a clear separation — the dismiss option is not buried, but it follows the sign-in options in reading order (intention → exit, not exit → intention).

Provider ordering: Clerk controls the order via dashboard config. UX recommendation: most universally available provider first (Google), least typical last (Magic Link). GitHub second for this product's technical-user audience.

No redesign required between Phase 1 and Phase 2. The dialog layout is identical — Clerk adds buttons.
If more than 4 providers are ever enabled, consider capping the visible list at 4 with a "More options" expander. At 5+ providers the dialog grows tall enough to risk clipping on small mobile viewports. Defer this decision to when it is needed.

Mobile rules

Bottom-anchored on mobile (slide up from floor). Centered on desktop. Same <dialog> element and same content — only the positioning changes via CSS (media query on the backdrop).

All touch targets: 44×44px minimum. Provider button: 44px min-height (padding 12px + font ~20px + padding 12px). "Continue without signing in": same. No element below 44px.

The drag handle (decorative bar at top) signals dismissibility to mobile users accustomed to bottom sheets. It is aria-hidden — the actual dismiss is the button below.

No horizontal padding less than 16px on 375px viewport. Font sizes slightly reduced vs. desktop but not below 9px for decorative elements or 11px for readable copy.

Accessibility Requirements
Element Requirement
<dialog> aria-modal="true" + aria-labelledby bound to H1 id. Screen readers announce "Sign in to sync your ledger, dialog."
Focus trap On dialog open, focus moves to the first provider button. Tab cycles within the dialog only (provider buttons → "Continue without signing in"). Shift+Tab reverses. No focus leaks to the backdrop or dashboard behind.
Escape key Closes dialog — same behavior as "Continue without signing in." Handled by the Radix Dialog primitive (not custom code).
Provider buttons Each has a descriptive aria-label ("Continue with GitHub"). Provider icon placeholder: aria-hidden="true". Min 44×44px touch target.
"Continue without signing in" Plain text label — no aria-label override needed. Min 44px height. Must not be a <a> styled as a button — must be a real <button> element.
Atmospheric eyebrow aria-hidden="true" — decorative framing only.
Drag handle (mobile) aria-hidden="true" — purely decorative. Not interactive.
Focus return On dialog close (dismiss or ESC), focus returns to the element that triggered the dialog (ᛟ avatar or upsell banner CTA). Radix Dialog handles this automatically.
Component Contract — For FiremanDecko

Component name: SignInDialog

Trigger: open: boolean prop — controlled externally by the parent (TopBar, upsell banner, Settings).

On dismiss: calls onClose() callback. Does NOT set upsell banner dismiss flag.

Providers: rendered dynamically from Clerk's enabled providers — no hardcoded list in this component. Provider order follows Clerk dashboard config.

Post-sign-in: Clerk handles the OAuth callback. On success, call onSignInSuccess() → check for anonymous data → trigger migration prompt if needed.

Desktop: centered in viewport (flexbox). Mobile (< 640px): anchored to bottom of viewport.

Backdrop: semi-transparent overlay. Click on backdrop does NOT close — only the "Continue without signing in" button and Escape key close the dialog.

Non-negotiable: "Continue without signing in" is a <button>, full width, min 44px height, always visible without scrolling (all viewport sizes from 375px up).
