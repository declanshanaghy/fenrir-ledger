Mobile rules

The sign-in card fills most of the viewport vertically on mobile. No horizontal max-width constraint — card uses full width with 16px padding.
"Continue without signing in" remains full-width — never collapses to a link.
TopBar is present and visible. The ᛟ avatar is still tappable (opens the upsell panel, though on this page it is less relevant — the sign-in CTA is already present).
Consider: on the /sign-in page, tapping the ᛟ avatar does nothing (or simply shows "You're already here to sign in"). Defer to FiremanDecko for the implementation decision.

Accessibility Requirements
Element Requirement
<main> (sign-in card) aria-labelledby bound to the H1 heading id. Screen readers announce the page landmark by heading name.
H1 heading One per page. Focusable on navigation for screen readers. Must be the first meaningful element inside <main>.
"Sign in to Google" button Full accessible label — no icon-only. Min 44px height touch target.
"Continue without signing in" button Same touch target as primary CTA. Must never be rendered as a link styled smaller than the primary button. The escape path is non-negotiable.
Atmospheric eyebrow aria-hidden="true" — decorative framing, not content the user needs to navigate by.
Feature list aria-label="What signing in gives you" on the container. Check icons: aria-hidden="true".
