A. Design Decisions

IssueRoot CauseFix

Pack member gap too large gap-3.5 + mt-0.5 on quote Reduce gap to 8px; align icon + name + role in one row
Gleipnir items tall font-body text-base (16px) Compress to 12px; reduce gap between items

F. Accessibility Requirements
ElementRequirementImplementation Note
Profile icons (rune glyphs) aria-hidden="true" on icon span; name/role text is adjacent and readable Icons are purely decorative; screen reader reads name + role from text
Fragment status icons aria-label on the status span: "fragment N found" or "fragment N not yet found" Do not rely on checkmark/lock glyph alone for state communication
Fragment II hidden button aria-label="The beard of a woman — easter egg trigger" so SR users can locate and activate Intentionally hidden from visual design, but must be keyboard accessible
Fragment dimming (unfound) opacity: 0.35 visual only; text content unchanged for SR Do not use aria-disabled or aria-hidden on unfound items
Pack quote truncation Full quote in title attribute of .member-quote element Allows hover/focus reveal of full text; tooltip pattern
Fragment count aria-live="polite" if it updates without page reload Currently reads on open only — no live update needed
Keyboard navigation Tab order: Header → Pack items → Gleipnir list (fragment II button in flow) → Close Fragment II must be in natural tab order
Focus ring All interactive elements (buttons, close) have visible focus ring Minimum 2px solid outline, offset 2px — matches existing pattern
Touch targets Close button min 44×44px; Pack icon-row min 36px height Profile icons not interactive — no touch target size required
