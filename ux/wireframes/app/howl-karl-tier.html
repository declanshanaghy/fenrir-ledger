Scenario 5 — Ragnarök Overlay Gate

Ragnarök fires at ≥5 urgent cards. Thrall users never see it. Karl users trigger it normally.

ᚱ Ragnarök Threshold — Tier Gating

The Ragnarök overlay is part of the Howl system. Its trigger condition checks tier before firing:

Thrall

Ragnarök never triggers, regardless of how many urgent cards exist.

tier !== 'karl' → skip Ragnarök check entirely.

No indication shown to Thrall users that this threshold exists — it's a Karl discovery.

Karl

Ragnarök fires normally when urgentCount ≥ 5.

tier === 'karl' && urgentCount >= 5 → trigger overlay.

Full Ragnarök experience unchanged from existing implementation.

Implementation: gate the urgentCount check in RagnarokContext behind tier check. See wireframes/notifications/ragnarok-threshold.html for overlay design.

Fake Sample Alert Content — Rules

The teaser must feel real enough to show the value of The Howl, but must use 100% hardcoded fake data.

RuleDetail
No real user data Teaser component receives no props from the user's card store. Fake issuer names and amounts are hardcoded strings in the component.
Representative variety Show 2–3 fake alerts: at least one fee alert (annual fee) and one promo alert (welcome bonus). Different urgency levels (7 days, 14 days, 30 days).
Real card brands OK Using real issuer names (Chase, Amex, Capital One) is fine — these are generic examples, not user data.
No accessibility exposure aria-hidden="true" on teaser list. Screen readers only see the overlay's dialog content.

Accessibility Requirements

ElementARIA / Keyboard requirement
Howl tab (Thrall) role="tab", aria-selected="true/false". Add aria-label="The Howl — Karl tier feature" to communicate gate to screen readers. The lock icon is decorative: aria-hidden="true".
Teaser list aria-hidden="true" on entire .teaser-list container. role="presentation". Content must not be reachable via Tab key (pointer-events: none not sufficient alone — add tabindex="-1" to all interactive elements within).
Upsell overlay role="dialog" (not modal — does not trap focus like a true modal; user can still Tab to other tabs). aria-label="Unlock The Howl". CTA button has descriptive label. "See all Karl features" link has visible text.
Upgrade CTA Minimum touch target 44×44px on mobile. Full-width button satisfies this on 375px viewport.
KARL badge Decorative text badge, aria-hidden="true" (the tab's aria-label covers the semantic meaning).
