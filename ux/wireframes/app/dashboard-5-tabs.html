Interaction Spec

Tab Classification — Status Routing

  • The Howl — status: fee_approaching, promo_expiring, overdue. Each card appears in exactly one status tab.
  • The Hunt — status: bonus_open. Previously lumped with active; now isolated.
  • Active — status: active only. Excludes bonus_open (which is The Hunt).
  • Valhalla — status: closed. Previously filtered out by getCards() — this filter must be removed/modified.
  • All — all 6 statuses combined. Each card appears here AND in its status tab.
  • Status computed by existing computeCardStatus() in card-utils.ts — no new computation needed.

Tab Persistence (localStorage)

  • Key: fenrir:dashboard-tab
  • Values: 'howl' | 'hunt' | 'active' | 'valhalla' | 'all'
  • On tab switch: localStorage.setItem('fenrir:dashboard-tab', tabId)
  • On mount: read localStorage.getItem('fenrir:dashboard-tab') to restore last selection.
  • First visit (no stored value) OR invalid stored value: default to The Howl if howlCards.length > 0, otherwise Active.
  • URL hash override (optional): /dashboard?tab=valhalla — if present, supersedes localStorage value. Used by sidebar Valhalla link.
  • Do NOT reset tab during session when cards update — only recalculate default on fresh mount.

Default Tab Selection Logic (pseudocode)

  • const stored = localStorage.getItem('fenrir:dashboard-tab');
    const urlTab = searchParams.get('tab');
    const initial = urlTab ?? stored ?? (howlCards.length > 0 ? 'howl' : 'active');
  • Validate initial is one of the 5 valid tab IDs before applying; fall back to 'howl' / 'active' logic if invalid.

Sidebar Valhalla Link Behavior

  • Removed: /valhalla route and page. The sidebar link no longer navigates to a separate page.
  • New behavior: Sidebar Valhalla link navigates to /dashboard?tab=valhalla (if not already on dashboard) OR dispatches a custom event fenrir:activate-tab with detail: { tab: 'valhalla' } if already on the dashboard page — engineer's choice of mechanism.
  • Active state: When the Valhalla tab is selected, sidebar "Valhalla" item shows active styling (bold + left border) simultaneously with "Cards" — both are active.
  • Keyboard: Sidebar link must be focusable and activatable via Enter / Space.

Header Card Count

  • The summary text (e.g. "11 cards total · 3 need attention") shows the full portfolio count including closed cards.
  • Previously getCards() filtered out closed cards — the count must now use all-statuses source.
  • Tab badges derive from their filtered subsets: howlCards.length, huntCards.length, activeCards.length, closedCards.length, allCards.length.
  • "N need attention" still refers to Howl count (fee_approaching + promo_expiring + overdue).

Keyboard Navigation (ARIA Tabs Pattern)

  • Tab bar: role="tablist". Each button: role="tab", aria-selected, aria-controls, id.
  • Panels: role="tabpanel", aria-labelledby, tabindex="0".
  • ArrowRight — focus + activate next tab (wraps around).
  • ArrowLeft — focus + activate previous tab (wraps around).
  • Home — focus + activate first tab (The Howl).
  • End — focus + activate last tab (All).
  • Tab from tablist — moves focus to active tab panel content.
  • All 5 tabs must be accessible at 375px mobile with default font size.

prefers-reduced-motion

  • The urgency dot on Howl cards has a pulsing animation — must be disabled when prefers-reduced-motion: reduce.
  • Tab panel transition (optional 150ms fade) — must be disabled when prefers-reduced-motion: reduce. Instant swap instead.
  • Implementation: @media (prefers-reduced-motion: reduce) { .urgency-dot { animation: none; } .tab-panel { transition: none; } }

Mobile Tab Bar (375px)

  • Tab bar: overflow-x: auto, -webkit-overflow-scrolling: touch, scrollbar hidden via scrollbar-width: none / ::-webkit-scrollbar { display: none }.
  • Min touch target per tab: 44px height (WCAG 2.5.5).
  • At 375px, approximately 2–2.5 tabs are fully visible — the rest scroll into view. This is expected and intentional.
  • No horizontal scrollbar shown. Users discover scrollability via partially-visible tabs.
  • Restore scroll position to selected tab on mount (use scrollIntoView({ behavior: 'smooth' }) or instant for reduced-motion).

Data Layer Changes Required (for FiremanDecko)

  • storage.ts getCards() line ~327: Remove or gate the c.status !== 'closed' filter. The Valhalla tab needs closed cards. Suggested: keep filter for existing consumers OR introduce getAllCards() that includes closed.
  • storage.ts getClosedCards(): Already exists — may be used directly for Valhalla tab.
  • Active tab change: activeCards must now filter to status === 'active' only (not bonus_open). Previously lumped together.
  • Hunt cards: New derived set — cards.filter(c => c.status === 'bonus_open').
  • Loki Mode: Preserved — continues to affect how cards render within their tabs.
  • Ragnarok Mode: Preserved — threshold logic unchanged, still based on Howl urgency count.

Component Architecture Suggestions (for FiremanDecko)

  • Extend existing DashboardTabs.tsx from 2 to 5 tabs — same ARIA tablist pattern.
  • Tab IDs: 'howl' | 'hunt' | 'active' | 'valhalla' | 'all' — add to existing union type.
  • Consider shadcn/ui Tabs (Tabs, TabsList, TabsTrigger, TabsContent) for ARIA compliance.
  • The Hunt card bonus bar: thin wrapper variant on existing CardChain component, add bonusBar prop.
  • Valhalla card: same CardChain component with status="closed" — already has dashed ring style presumably.
  • All tab status pill: thin overlay component; CardChain can accept showStatusPill boolean prop.
  • localStorage hook: useLocalStorage('fenrir:dashboard-tab', defaultTab) — can be extracted as a utility hook.
