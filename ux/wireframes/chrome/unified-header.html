Unified Header — Issue #1853

Marketing and ledger headers are inconsistent in height, logo size, typography, and spacing. This wireframe defines a single unified header shell used by both surfaces. The ledger layout is the reference (compact 48px). Marketing keeps its CTA + inline theme toggle; Ledger keeps its avatar + Karl/Trial badge cluster.

1. Before / After — Height Alignment

Key change: Marketing navbar shrinks from 64px (h-16) to 48px (h-12)

Both headers must render at exactly 48px. All padding, logo sizes, and control heights are constrained to fit cleanly. No other visible element in the header changes — only the shell height and logo text size.

6. Accessibility Requirements (WCAG 2.1 AA)

Both header variants must meet these criteria

  • Skip nav link — sr-only, visible on :focus. Target: #main-content. Both variants (marketing currently missing this).
  • role="banner" — header element wraps entire nav. Both variants.
  • aria-label on nav — "Marketing site navigation" for center nav, "Mobile navigation" for overlay nav.
  • aria-expanded — hamburger button reports open/closed state.
  • aria-controls — hamburger controls the overlay id.
  • aria-current="page" — active nav link.
  • Touch targets ≥ 44×44px — hamburger, close button, avatar button, CTA all satisfy WCAG 2.5.5.
  • Escape closes overlay — focus returns to hamburger on close.
  • Focus trap in overlay — auto-focus close button on open; Tab cycles within overlay.
  • Body scroll lock — overflow:hidden on body when overlay open.
  • Logo aria-label — "Fenrir Ledger — home" (or "go to home" for ledger context).
