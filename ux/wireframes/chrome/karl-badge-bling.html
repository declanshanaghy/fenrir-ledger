A — Rune Selection

Issue #1928 requires different runes on left vs right. Both currently use ᚷ (Gebo). The new pair must be visually distinct and thematically appropriate for a paid-tier indicator.

Chosen Pair

PositionRuneNameMeaningVisual shapeRationale
Left ᚠ Fehu Wealth / prosperity F-like fork shape — open upward branches Already used as a wealth marker in .karl-bling-btn::before (consistent with existing Karl rune grammar). Directly evokes the paid-tier concept.
Right ᛟ Othala Heritage / ancestral home / nobility Diamond-over-base — closed, grounded shape Used in the Fenrir logo (ᛟ). Strong visual contrast to Fehu (open vs closed geometry). Evokes lineage and belonging — fitting for a subscriber identity badge.

Rune grammar note: The existing css uses ᚠ Fehu (btn::before) and ᚱ Raidho (btn::after) for buttons — the badge uses a different pair to feel distinct. ᚷ Gebo (current badge rune) is retired from the badge slot.

I — FiremanDecko Implementation Notes

  • Files changed: KarlBadge.tsx (rune characters only) + karl-bling.css (padding, font-size, hover rule)
  • No new classes needed: All changes are to existing selectors. The :hover rule for .karl-bling-badge is new but uses existing CSS custom properties.
  • Gold glow value: Issue #1928 suggests hover:shadow-[0_0_12px_rgba(201,146,10,0.5)] — use this as the primary shadow, optionally adding a tighter inner glow: 0 0 4px rgba(201,146,10,0.7). Tweak opacity to match the visual weight of the existing karl-badge-pulse animation.
  • Taller badge: Achieved via padding change (0.1rem → 0.25rem vertical). Do not use min-height — let the content drive height naturally.
  • Rune font-size: Increase .karl-badge-rune from 0.7rem to 0.85rem. This applies to both Karl and trial tiers (structural parity).
  • Component comment: Update the JSDoc comment in KarlBadge.tsx from "Runic accents (ᚷ Gebo)" to "Runic accents (ᚠ Fehu left, ᛟ Othala right)".
  • Tests to update: Check __tests__/karl-bling/karl-badge.test.tsx — any test asserting the ᚷ character will need updating to ᚠ and ᛟ respectively.
  • Light and dark themes: The CSS cascade uses CSS custom properties (--karl-gold, --karl-shadow-*) that are already theme-aware. No theme-specific overrides needed.
