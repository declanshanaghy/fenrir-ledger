8. Consistent Row Layout Pattern

All action rows follow this exact pattern
┌─────────────────────────────────────────────────────┐
│  px-4                                          px-4  │
│  ┌────┐  ┌──────────────────────────────────────┐   │
│  │icon│  │ label text                           │   │ ← 44px min height
│  │16px│  │ font-body, text-muted-foreground      │   │   py-3 (12px top+bottom)
│  └────┘  └──────────────────────────────────────┘   │
│    ↑         ↑                                       │
│   gap-2 (8px)                                        │
└─────────────────────────────────────────────────────┘

flex items-center gap-2 px-4 py-3 (min-height 44px)
icon: h-4 w-4, flex-shrink: 0
NO right-side elements (no chevrons, no secondary icons)
This pattern is applied identically to: Theme row, Settings row, Sign out row.
Theme row: the <ThemeToggle variant="icon" /> component renders the icon. Wrap it in the flex row — the toggle's own icon IS the left icon.

9. Accessibility Requirements

Requirement Implementation
Min touch target 44×44px style={{ minHeight: 44 }} on each menu-item button
Icon-only elements have text alternative Theme icon: aria-label="Toggle theme, current: [Light|Dark|System]"
Keyboard navigation Esc closes dropdown, Tab moves focus, Enter/Space activates
ARIA roles Container: role=menu. Rows: role=menuitem. Header: aria-hidden=true.
Focus management On open: focus first menuitem (Theme). On close: return focus to trigger.
Decorative icons All lucide icons: aria-hidden=true (they accompany visible text labels)
