LedgerShell — Slim Top Bar + Mobile Bottom Tab Bar

Wireframe for Issue #372 · ux/wireframes/chrome/ledger-shell.html · Structural layout only — no theme styling

Purpose: A new layout shell for all /ledger/* routes that replaces the existing AppShell. The marketing site ( /, /features, /pricing, /blog, /about ) is unaffected and retains its full marketing navbar and footer.

Key differences from the old AppShell:
• Slim top bar: 48px instead of 56px — removes "Credit Card Tracker" subtitle, condenses brand mark
• Sidebar: 220px (was 272px) — fits the condensed app zone feel
• Mobile: sidebar replaced by fixed bottom tab bar — no hamburger, no overlay
• Center slot: "← Back to site" link — contextual escape hatch to /
• No marketing footer inside /ledger/* — full-viewport app experience

Breakpoint: 768px is the single toggle point. Below it: mobile mode (bottom tabs). Above it: desktop mode (sidebar). This matches the existing breakpoint convention in the codebase.
No intermediate "tablet" mode: At 768–1024px, the sidebar is visible and the bottom tab bar is hidden. The sidebar at 220px leaves ~500–800px for content — sufficient for the card grid at 2-column layout.
CSS implementation: .ledger-sidebar { display: none; } / .bottom-tab-bar { display: none; } with @media (min-width: 769px) / @media (max-width: 768px) toggles. Or use Tailwind: hidden md:flex / flex md:hidden.

Interaction Spec — LedgerShell
Theme toggle
Clicking the theme toggle icon button cycles between light ↔ dark mode. It uses the same ThemeToggle component as the marketing site — no behavioral difference. The visual icon renders inline (no label text visible). The component is stateless from the shell's perspective — theme state lives in ThemeProvider.

Avatar / user dropdown
Identical to the existing TopBar avatar behavior (see topbar.html for full spec). The only change is the visual size: avatar is 28px visual (down from 32px) to match the slimmer bar. Touch target remains 44×44px. Anonymous state shows the upsell prompt panel. Signed-in state shows the user dropdown with Sign Out.

"Back to site" (desktop)
Standard <a href="/"> link — browser navigation. No custom JS needed. On mobile this becomes the back icon button. Both navigate to /.

Sidebar active state (desktop)
The sidebar nav item matching pathname receives aria-current="page" and the active CSS class (left border accent + bold). This is driven by the router's current pathname — no JS state needed beyond what usePathname() provides. On route change, the active item updates automatically.

Bottom tab bar active state (mobile)
Same mechanism: usePathname() comparison. Active tab receives aria-current="page" + accent styling. No animation between tabs — instant route transition is sufficient. If a tab route is the same as the current route, tapping it is a no-op (or optionally scrolls to top).

Shell persistence across /ledger/* routes
The LedgerShell is the layout component for the (app) route group in Next.js. It renders once and its children (the page-level content) swap on navigation. The top bar, sidebar, and bottom tab bar never unmount between /ledger/* routes. Navigating from /ledger to /ledger/valhalla: only the main content area re-renders.

Scroll behavior
The top bar is position: sticky; top: 0. The sidebar is position: sticky; top: 48px; height: calc(100vh - 48px). The <main> content area scrolls independently (overflow-y: auto). The bottom tab bar is position: fixed; bottom: 0 — unaffected by content scroll.

No marketing footer
The LedgerShell does not render the marketing footer. The app is full-viewport — the bottom of the content area is the bottom of the page (or the fixed bottom tab bar on mobile). Any app-specific footer content (copyright in Settings, etc.) is inlined into the page content, not the shell.
