Video Portal Interaction

State machine: thumbnail → playing

  • State A — Thumbnail: img (YouTube hqdefault) + gold play overlay. role="button", tabindex="0".
  • On click/keydown Enter/Space: → State B
  • State B — Playing: Replace img + overlay with <iframe> src="https://www.youtube.com/embed/QRg_8NNPTD8?autoplay=1&rel=0". title="Heilung — Krigsgaldr LIFA". allow="autoplay; encrypted-media". allowfullscreen.
  • Fallback: If JS disabled → portal frame is an <a> link to youtu.be/QRg_8NNPTD8 target="_blank".

Implementation pattern (React)

  • Local state: const [playing, setPlaying] = useState(false)
  • Render: {playing ? <iframe …/> : <ThumbnailOverlay onClick={() => setPlaying(true)} />}
  • On modal close: reset playing to false (stops video).

Accessibility Notes (WCAG 2.1 AA)

New/changed requirements vs Issue #955

  • Algiz close button: aria-label="Close — return from the wolf's hall". The ᛉ glyph is not announced — label provides accessible name.
  • Norse title: aria-label="Heyra Stríðsgaldr — Hear the War-Chant" — English translation in aria-label for screen readers.
  • Rune bands: aria-hidden="true" — decorative. No linguistic content.
  • Wiki links: Each <a> has aria-label="{term} on Wikipedia". Opens in new tab — indicated via aria-label ("on Wikipedia" is idiomatic; "opens in new tab" may be added per team convention).
  • Video portal: role="button" + tabindex="0" on thumbnail container (State A). aria-label="Watch Heilung — Krigsgaldr LIFA on YouTube".
  • Gold terms (non-linked): <span class="gold-term"> — color only, no semantic role. Ensure gold (#c9920a) on dark bg passes WCAG AA large text. Supplementary meaning conveyed by context (body copy), not color alone.
  • Scroll: modal overflow-y: auto — ensure scroll region is keyboard accessible. Add tabindex="0" to modal scrollable div if needed per Firefox behavior.
  • Wolf seal: aria-label="Fenrir seal — The wolf remembers what the world forgot" on seal paragraph. The rune glyphs (ᚠᛖᚾᚱᛁᚱ) are read literally by screen readers — acceptable since they precede/follow the readable English text.
