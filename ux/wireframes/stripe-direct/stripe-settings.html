Accessibility Requirements
Element Requirement
Settings card region role="region" + aria-labelledby bound to section heading. Screen readers announce "Subscription" on region entry.
Status badge Text content is sufficient -- no icon-only state. Screen readers announce "Thrall" or "Karl" naturally.
Subscribe button Full text label includes price. Min 44px touch target. Focus visible.
Cancel button aria-label="Cancel subscription" for disambiguation from other cancel buttons on page.
Atmospheric subhead aria-hidden="true" -- decorative Norse copy, not functional content.
Interaction Specification -- State Machine

State Diagram (Mermaid -- render in wireframes.md)

stateDiagram-v2
    [*] --> Thrall: No subscription
    Thrall --> CheckoutRedirect: Subscribe clicked
    CheckoutRedirect --> Karl: Stripe webhook confirms
    CheckoutRedirect --> Thrall: User abandons checkout
    Karl --> Canceled: Cancel via Stripe Portal
    Canceled --> Karl: Resubscribe before period end
    Canceled --> Thrall: Period ends (webhook)
    Karl --> Karl: Renewal (automatic)

Data flow

  1. Component mounts: GET /api/stripe/status to fetch current tier + status.
  2. Subscribe: POST /api/stripe/checkout-session -> Stripe Checkout URL -> redirect.
  3. Manage/Cancel: POST /api/stripe/portal-session -> Stripe Portal URL -> window.open().
  4. Resubscribe: same as Subscribe (POST /api/stripe/checkout-session).
  5. Webhook: /api/stripe/webhook processes checkout.session.completed, customer.subscription.updated, customer.subscription.deleted.
