Interaction Specification -- SealedRuneModal Flow

Trigger

User (Thrall tier) clicks on a feature gated by requireKarl feature flag. The component checks subscription status. If not Karl, renders SealedRuneModal instead of the feature.

Subscribe flow (Mermaid -- render in wireframes.md)

sequenceDiagram
    participant U as User
    participant M as SealedRuneModal
    participant E as EmailModal
    participant A as /api/stripe/checkout-session
    participant S as Stripe Checkout

    U->>M: Clicks locked feature
    M->>M: Renders modal (Thrall gate)
    alt Authenticated user
        U->>M: Clicks "Subscribe for $3.99/month"
        M->>A: POST {priceId, email: googleEmail}
        A-->>M: {checkoutUrl}
        M->>S: Redirect to Stripe Checkout
    else Anonymous user
        U->>M: Clicks "Subscribe for $3.99/month"
        M->>E: Opens email collection modal
        U->>E: Enters email + submits
        E->>A: POST {priceId, email: userEmail}
        A-->>E: {checkoutUrl}
        E->>S: Redirect to Stripe Checkout
    end
    S-->>U: Checkout complete -> success_url

Dismiss behavior

  • "Not now" button: closes modal, returns to underlying view.
  • X close button: same behavior as "Not now".
  • Backdrop click: same behavior as "Not now".
  • Escape key: closes modal.
  • No permanent dismiss flag -- modal reappears on next locked feature access.
