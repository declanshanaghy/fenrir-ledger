Accessibility Requirements
Element Requirement
Modal dialog role="dialog" + aria-modal="true" + aria-labelledby bound to heading. Focus trapped inside modal.
Email input type="email", autocomplete="email", associated <label>, aria-describedby for hint/error text. Receives focus on modal open.
Error message role="alert" for live announcement. Input gets aria-invalid="true" when in error state.
Submit button Full text label. Disabled state communicated via aria-disabled="true" (not just visual opacity).
Loading indicator aria-live="polite" -- screen readers announce the loading text without interrupting.
Escape key Closes modal (same as Cancel). Disabled during loading state.

Email validation rules

  • Required field -- cannot be empty.
  • Must match email format (HTML5 type="email" + JS regex for edge cases).
  • No maximum length enforced on the client (Stripe handles server-side validation).
  • Validation runs on blur and on submit.
  • Error clears when user starts typing (onChange).

Error states

  • Empty: "Please enter your email address."
  • Invalid format: "Please enter a valid email address."
  • API error: "Something went wrong. Please try again." (generic -- do not expose Stripe errors).
