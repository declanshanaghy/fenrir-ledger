## Description

The All-Father's Decree renders as a wall of raw monospace text inside a collapsible NorseTablet. It deserves the ceremony of a royal decree — carved in stone, handed through Huginn and Muninn to the agent.

## Current Behavior

The NorseTablet component (LogViewer.tsx:344) renders the raw prompt as a single div.

4 — Section Body: Inline Formatting

The raw prompt text is preserved in full — inline formatting is applied on top of it, not instead of it. No content is removed or truncated.

4a — Code Spans

Shell commands, file paths, and backtick-wrapped terms are rendered as code spans:

Run git branch --show-current and node -v and ls package.json

Verify at: development/frontend/package.json

Match pattern: backtick-wrapped text `...` → <code class="decree-body-code">. Also apply to patterns matching file paths like path/to/file.ts and shell commands on their own line.

4b — Norse Term Wikipedia Links

Terms in WIKI_LINKS are linked in gold wherever they appear in any section body:

From the roots of Yggdrasil, across the Bifröst, into Valhalla and the Nine Worlds

Links open in target="_blank" rel="noopener noreferrer". Link text is gold (var(--gold)), underlined. The WIKI_LINKS map in constants.ts is the single source of truth — no hardcoding in component.
Matching is case-sensitive and whole-word. "Bifröst" should not match "Bifröst-adjacent". Apply all WIKI_LINKS entries on every render of DecreeSectionBody.

4c — Code Block Formatting

Multi-line command blocks (lines starting with cd, git, gh, bash, npm, node, or indented by 4+ spaces) are rendered as code blocks:

cd /workspace/repo && git add -A && git commit -m 'wip: ...' && git push origin ux/issue-1020-decree-inscription cd /workspace/repo && bash quality/scripts/verify.sh --step tsc
These blocks render inside the section body — they preserve line breaks and use monospace. They should NOT be syntax-highlighted (no color). Existing raw text content stays but code lines are wrapped in <code class="decree-body-block">.
