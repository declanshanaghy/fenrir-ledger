CSS Implementation Strategy

ConcernApproach
Theme selector Add data-theme="light" to <html> for light mode. Default (no attribute) = dark.
CSS variables Existing :root in index.css = dark tokens (keep as-is).
Add [data-theme="light"] block with light overrides (derived from globals.css :root).
Persistence localStorage key: fenrir-monitor-theme. Values: "light" or "dark" (default).
JS/React hook New hook useTheme() in src/hooks/useTheme.ts.
Reads localStorage on mount, writes on toggle, applies data-theme attribute.
Component New component src/components/ThemeSwitcher.tsx.
Renders two <button> elements inside a <div role="group">.
Transition transition: background 0.2s, color 0.2s, border-color 0.2s on body.
Avoids property wildcard (transition: all) — only theme-relevant properties.

6. Accessibility Requirements

CriterionRequirement
WCAG 2.5.5 Min touch target: 44×44px. Each theme button must meet this on mobile.
WCAG 1.4.3 / AA All text/icon pairs in both themes must achieve ≥4.5:1 contrast. See token map above.
Keyboard Both buttons must be Tab-reachable and activated via Enter/Space.
ARIA role="group" + aria-label="Choose theme" on wrapper. aria-pressed="true/false" on each button.
Focus ring Visible focus ring on both buttons (2px gold outline, offset 2px). No outline: none.
Reduced motion CSS transition on body/html can remain (background/color changes are functional, not decorative). No animation keyframes needed for toggle itself.
Screen reader On activation: aria-pressed update is sufficient. No additional aria-live region needed since the entire page visually updates (screen reader users perceive context change).
