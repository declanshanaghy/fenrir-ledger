<div style={{padding:'40px 0',textAlign:'center'}}>
<blockquote style={{margin:'0 auto',maxWidth:'760px',fontWeight:300,fontStyle:'italic',fontSize:'0.95rem',color:'#e8e4d4',lineHeight:1.8,borderLeft:'2px solid #8a6a00',paddingLeft:'24px',textAlign:'left'}}>
The wolf does not howl without cause.

We built an autonomous agent orchestration system on top of <code style={{fontFamily:"'JetBrains Mono', monospace",fontSize:'0.85em',color:'#5b9ec9'}}>depot claude</code>.
Multiple agents — engineers, testers, designers — each running in their own sandbox,
each handing off to the next via git commits and GitHub issue comments.
The architecture is sound. The wolf runs.

But the chain that binds us is not the work itself — it is the tooling around it.
We have fought these battles long enough to know the difference between friction
that sharpens and friction that bleeds. This is the latter.

We are writing this because we *want* to stay. Depot's sandbox model is the
closest thing to what we need. But a wolf that cannot see its pack, cannot hear
when the hunt ends, and must claw through the underbrush for every scrap of
information — that wolf will eventually find another forest.

If these gaps remain, we will be forced to evaluate alternatives — raw EC2
sandboxes, modal.com, or rolling our own container orchestration. None of those
options are as elegant as what Depot *could* be. That's what makes this
worth writing.

What follows are the thorns we pulled from our paws. We offer them not as complaints,
but as prey laid at your door — problems worth hunting, before the wolf moves on.
</blockquote>
</div>

<div className="rune-hr"><span>ᚠ ᛖ ᚾ</span></div>

<div style={{padding:'24px 0 64px'}}>

<div style={{background:'#0f1018',border:'1px solid #1e2235',borderLeft:'3px solid #c94a0a',borderRadius:'3px',padding:'40px 36px',marginBottom:'32px',boxShadow:'0 4px 24px rgba(0,0,0,0.4)'}}>
<span style={{fontFamily:"'JetBrains Mono', monospace",fontSize:'0.68rem',letterSpacing:'0.3em',textTransform:'uppercase',color:'#c94a0a',marginBottom:'8px',display:'block'}}>Grievance I</span>
<h2 style={{fontFamily:"'Cinzel', serif",fontWeight:700,fontSize:'clamp(20px, 3vw, 26px)',color:'#e8e4d4',letterSpacing:'0.06em',margin:'0 0 8px'}}>No Simple Way to Retrieve a Session Log</h2>
<p style={{fontStyle:'italic',fontSize:'0.95rem',color:'#c9920a',margin:'0 0 16px',lineHeight:1.6}}>There is no <code>depot claude session-log &lt;id&gt;</code> command.</p>
<p style={{color:'#e8e4d4',fontSize:'0.9rem',lineHeight:1.75,margin:'0 0 16px'}}>To get the transcript of what an agent did in a sandbox, we must: search <code>~/.claude/projects/&lt;hash&gt;/</code> for JSONL files, filter by file size (Depot sessions are small; local conversations are large), grep inside each file for the session ID or branch name, and if not found locally, run <code>depot claude --wait --resume &lt;id&gt;</code> to trigger a download, then parse the JSONL with <code>jq</code>.</p>
<p style={{color:'#e8e4d4',fontSize:'0.9rem',lineHeight:1.75,margin:'0 0 16px'}}>We wrote a 90-line bash script to automate this. It should be one command.</p>

<span style={{display:'block',fontFamily:"'JetBrains Mono', monospace",fontSize:'0.68rem',letterSpacing:'0.15em',textTransform:'uppercase',marginBottom:'12px',color:'#0a8c6e'}}>What we want</span>
<div className="code-block">
<pre>{`# Get the full session transcript
depot claude session-log <session-id>

# Raw JSONL for piping
depot claude session-log <session-id> --format jsonl

# Just assistant messages, human-readable
depot claude session-log <session-id> --format text`}</pre>
</div>

<span style={{display:'block',fontFamily:"'JetBrains Mono', monospace",fontSize:'0.68rem',letterSpacing:'0.15em',textTransform:'uppercase',marginBottom:'12px',color:'#c94a0a'}}>What we have to do instead</span>
<div className="code-block">
<pre>{`# Our actual script — this is what "get a log" looks like today
JSONL_FILE=$(find ~/.claude/projects/ -name "*.jsonl" -size -500k \\
  -exec grep -l "$SESSION_ID" {} \\;)

if [ -z "$JSONL_FILE" ]; then
  depot claude --wait --resume "$SESSION_ID" --org "$ORG_ID" >/dev/null 2>&1
  JSONL_FILE=$(find ~/.claude/projects/ -name "*.jsonl" -size -500k \\
    -exec grep -l "$SESSION_ID" {} \\;)
fi

cat "$JSONL_FILE" | jq -r '...' # 40 more lines of jq incantations`}</pre>
</div>

<p style={{fontStyle:'italic',fontSize:'0.88rem',color:'#e8e4d4',marginTop:'20px',paddingTop:'16px',borderTop:'1px solid #1e2235'}}>The wolf should not need to dig through filesystem internals to see what its pack did.</p>
</div>

<div style={{background:'#0f1018',border:'1px solid #1e2235',borderLeft:'3px solid #c94a0a',borderRadius:'3px',padding:'40px 36px',marginBottom:'32px',boxShadow:'0 4px 24px rgba(0,0,0,0.4)'}}>
<span style={{fontFamily:"'JetBrains Mono', monospace",fontSize:'0.68rem',letterSpacing:'0.3em',textTransform:'uppercase',color:'#c94a0a',marginBottom:'8px',display:'block'}}>Grievance II</span>
<h2 style={{fontFamily:"'Cinzel', serif",fontWeight:700,fontSize:'clamp(20px, 3vw, 26px)',color:'#e8e4d4',letterSpacing:'0.06em',margin:'0 0 8px'}}>CLI Defaults to Pager — Hostile to Automation</h2>
<p style={{fontStyle:'italic',fontSize:'0.95rem',color:'#c9920a',margin:'0 0 16px',lineHeight:1.6}}><code>depot claude list-sessions</code> opens an interactive pager by default. This is death for automation.</p>
<p style={{color:'#e8e4d4',fontSize:'0.9rem',lineHeight:1.75,margin:'0 0 16px'}}>Every script that calls <code>list-sessions</code> must pass <code>--output json</code> to avoid hanging on a pager that will never receive a keypress. There is no environment variable (<code>DEPOT_PAGER</code>, <code>DEPOT_NO_PAGER</code>, <code>NO_PAGER</code>) to disable this globally.</p>
<p style={{color:'#e8e4d4',fontSize:'0.9rem',lineHeight:1.75,margin:'0 0 16px'}}>Even with <code>--output json</code>, the command prefixes a pagination token line before the JSON array, requiring an additional <code>sed -n '/^\[/,$ p'</code> to extract valid JSON. The pagination token should not be mixed into the JSON output — put it in stderr, a header, or a separate <code>--next-token</code> flag.</p>

<span style={{display:'block',fontFamily:"'JetBrains Mono', monospace",fontSize:'0.68rem',letterSpacing:'0.15em',textTransform:'uppercase',marginBottom:'12px',color:'#0a8c6e'}}>What we want</span>
<div className="code-block">
<pre>{`# Environment variable to disable pager globally
export DEPOT_PAGER=off

# Or at minimum, detect non-TTY and disable automatically
depot claude list-sessions | jq '.[0].id'   # should just work`}</pre>
</div>

<span style={{display:'block',fontFamily:"'JetBrains Mono', monospace",fontSize:'0.68rem',letterSpacing:'0.15em',textTransform:'uppercase',marginBottom:'12px',color:'#c94a0a'}}>What we have to do instead</span>
<div className="code-block">
<pre>{`depot claude list-sessions --output json 2>/dev/null \\
  | sed -n '/^\\[/,$ p' \\
  | jq '.[0].id'`}</pre>
</div>

<p style={{fontStyle:'italic',fontSize:'0.88rem',color:'#e8e4d4',marginTop:'20px',paddingTop:'16px',borderTop:'1px solid #1e2235'}}>The wolf does not wait for permission to read its own rune log.</p>
</div>

<div style={{background:'#0f1018',border:'1px solid #1e2235',borderLeft:'3px solid #c94a0a',borderRadius:'3px',padding:'40px 36px',marginBottom:'32px',boxShadow:'0 4px 24px rgba(0,0,0,0.4)'}}>
<span style={{fontFamily:"'JetBrains Mono', monospace",fontSize:'0.68rem',letterSpacing:'0.3em',textTransform:'uppercase',color:'#c94a0a',marginBottom:'8px',display:'block'}}>Grievance III</span>
<h2 style={{fontFamily:"'Cinzel', serif",fontWeight:700,fontSize:'clamp(20px, 3vw, 26px)',color:'#e8e4d4',letterSpacing:'0.06em',margin:'0 0 8px'}}>Sessions Are Resumable — But We Want One-Shot</h2>
<p style={{fontStyle:'italic',fontSize:'0.95rem',color:'#c9920a',margin:'0 0 16px',lineHeight:1.6}}>The session model assumes you'll come back. We never do.</p>
<p style={{color:'#e8e4d4',fontSize:'0.9rem',lineHeight:1.75,margin:'0 0 16px'}}>The <code>depot claude</code> session model is built around resumption — start a session, leave, come back, resume. This is useful for interactive work. But our use case is <strong>fire-and-forget agent dispatch</strong>: launch an agent with a prompt, it does its work, it's done. We never resume it. We just want to know when it finishes.</p>
<p style={{color:'#e8e4d4',fontSize:'0.9rem',lineHeight:1.75,margin:'0 0 16px'}}>The <code>--wait</code> flag blocks until completion, which helps. But there's no lightweight way to check "is this session done yet?" without either running <code>--wait --resume</code> (blocks and downloads the full session) or listing all sessions and filtering (paginated, pager-hostile — see Grievance II).</p>

<span style={{display:'block',fontFamily:"'JetBrains Mono', monospace",fontSize:'0.68rem',letterSpacing:'0.15em',textTransform:'uppercase',marginBottom:'12px',color:'#0a8c6e'}}>What we want</span>
<div className="code-block">
<pre>{`# Simple status check — returns immediately
depot claude session-status <session-id>
{ "id": "...", "status": "completed", "exit_code": 0, "duration": "3m42s" }

# Exit-code-based check for scripting
depot claude session-status <session-id> --quiet
# Exit 0 = completed, Exit 1 = running, Exit 2 = failed`}</pre>
</div>

<span style={{display:'block',fontFamily:"'JetBrains Mono', monospace",fontSize:'0.68rem',letterSpacing:'0.15em',textTransform:'uppercase',marginBottom:'12px',color:'#c94a0a'}}>What we have to do instead</span>
<div className="code-block">
<pre>{`# Option A: Block forever
depot claude --wait --resume "$SESSION_ID" --org "$ORG_ID"

# Option B: Poll the session list (paginated, pager-hostile)
while true; do
  STATUS=$(depot claude list-sessions --output json 2>/dev/null \\
    | sed -n '/^\\[/,$ p' \\
    | jq -r ".[] | select(.id == \\"$SESSION_ID\\") | .status")
  [ "$STATUS" = "completed" ] && break
  sleep 30
done`}</pre>
</div>

<p style={{fontStyle:'italic',fontSize:'0.88rem',color:'#e8e4d4',marginTop:'20px',paddingTop:'16px',borderTop:'1px solid #1e2235'}}>The wolf needs to watch all its pups at once, not sit beside each one until it sleeps.</p>
</div>

<div style={{background:'#0f1018',border:'1px solid #1e2235',borderLeft:'3px solid #c94a0a',borderRadius:'3px',padding:'40px 36px',marginBottom:'32px',boxShadow:'0 4px 24px rgba(0,0,0,0.4)'}}>
<span style={{fontFamily:"'JetBrains Mono', monospace",fontSize:'0.68rem',letterSpacing:'0.3em',textTransform:'uppercase',color:'#c94a0a',marginBottom:'8px',display:'block'}}>Grievance IV</span>
<h2 style={{fontFamily:"'Cinzel', serif",fontWeight:700,fontSize:'clamp(20px, 3vw, 26px)',color:'#e8e4d4',letterSpacing:'0.06em',margin:'0 0 8px'}}>No Backchannel for Session Completion</h2>
<p style={{fontStyle:'italic',fontSize:'0.95rem',color:'#c9920a',margin:'0 0 16px',lineHeight:1.6}}>When an agent finishes, nothing happens on our end. No webhook. No callback. No event.</p>
<p style={{color:'#e8e4d4',fontSize:'0.9rem',lineHeight:1.75,margin:'0 0 16px'}}>Our current architecture is "fire-and-forget, then the human checks manually." This works, but it means the human is the event loop. The wolf is leashed to the handler's schedule.</p>
<p style={{color:'#e8e4d4',fontSize:'0.9rem',lineHeight:1.75,margin:'0 0 16px'}}>Our best workaround is to run <code>depot claude --wait --resume</code> as a background process and rely on our CLI framework's task notification system. But this means holding an open process per session, which doesn't scale to 5+ parallel agent chains.</p>

<span style={{display:'block',fontFamily:"'JetBrains Mono', monospace",fontSize:'0.68rem',letterSpacing:'0.15em',textTransform:'uppercase',marginBottom:'12px',color:'#0a8c6e'}}>What we want — webhooks</span>
<div className="code-block">
<pre>{`# Register a webhook for session lifecycle events
depot claude webhooks add \\
  --url "https://our-api.example.com/depot/callback" \\
  --secret "$WEBHOOK_SECRET" \\
  --events session.completed,session.failed

# Signed payload (HMAC-SHA256):
{
  "event": "session.completed",
  "session_id": "issue-199-step2-loki-fe3adaf1",
  "org_id": "pqtm7s538l",
  "exit_code": 0,
  "duration_seconds": 222,
  "repository": "declanshanaghy/fenrir-ledger",
  "branch": "fix/issue-199-cards-header-text"
}`}</pre>
</div>

<span style={{display:'block',fontFamily:"'JetBrains Mono', monospace",fontSize:'0.68rem',letterSpacing:'0.15em',textTransform:'uppercase',marginBottom:'12px',color:'#0a8c6e'}}>Or even simpler — per-session callback</span>
<div className="code-block">
<pre>{`depot claude \\
  --session-id "issue-199-step2-loki" \\
  --on-complete "https://our-api.example.com/depot/done" \\
  -p "..."`}</pre>
</div>

<p style={{fontStyle:'italic',fontSize:'0.88rem',color:'#e8e4d4',marginTop:'20px',paddingTop:'16px',borderTop:'1px solid #1e2235'}}>The wolf pack hunts in parallel. When one brings down its quarry, the others must know — without stopping to look over their shoulders.</p>
