        <div class="work-card">
          <div class="work-body">
            <p>Fixed 4 flaky CI tests across two test suites. Root causes: CSS comments in <span class="mono">globals.css</span> claimed hex values that didn't match actual HSL computations, and <span class="mono">next-themes</span> applies the theme class via a blocking script before React mounts &mdash; <span class="mono">page.evaluate()</span> runs too late.</p>
          </div>

          <div class="bug-box">
            <div class="bug-label">&#x1F41A; Easter Egg Color Assertions</div>
            <div class="bug-text">Konami Howl, Loki Mode, and Forgemaster modal tests asserted exact RGB values from inaccurate CSS comments. Fixed by replacing with visibility assertions.</div>
          </div>

          <div class="bug-box">
            <div class="bug-label">&#x1F41A; TC-TF-019 Dark Mode Test</div>
            <div class="bug-text">page.evaluate() sets localStorage after hydration — too late for next-themes. Fixed with addInitScript() which runs before page load.</div>
          </div>

          <div class="code-block">
            <pre><span class="cc">// Before (fails — runs after hydration):</span>
<span class="cr">await page.evaluate(() => localStorage.setItem(key, "dark"));</span>
<span class="cr">await page.reload();</span>

<span class="cc">// After (works — runs before page load):</span>
<span class="ca">await page.addInitScript((key) => {</span>
<span class="ca">  localStorage.setItem(key, "dark");</span>
<span class="ca">}, THEME_STORAGE_KEY);</span>
<span class="ca">await page.goto("/");</span></pre>
          </div>

          <div class="file-chips">
            <span class="chip chip-mod">&#x25C8; quality/test-suites/easter-eggs/easter-eggs.spec.ts</span>
