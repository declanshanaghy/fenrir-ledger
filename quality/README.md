## Test Architecture

### Three-Layer Pyramid

| Layer | Runner | Tests | Speed | Purpose |
|-------|--------|-------|-------|---------|
| **Unit** | Vitest | ~500+ | ~2s | Pure logic, utilities, helpers |
| **Integration** | Vitest + happy-dom | ~300+ | ~3s | API routes, component render, hooks |

## E2E Test Strategy (FiremanDecko ↔ Loki)

### FiremanDecko (Implementation)

**Responsibility:** Write and maintain E2E tests that match critical user journeys.

- **Only test:** OAuth flows, Stripe integration, card CRUD, data persistence, multi-page navigation, a11y gates
- **Never test:** Static content, CSS measurements, regression suites, issue-scoped directories
- **After each feature:** Create/update E2E suite under `quality/test-suites/<feature>/`
- **Max 8 tests per suite** — if you hit 15, split by sub-feature or move to integration tests
- **Incremental commits** every 1-3 files changed: `wip: <what> — Ref #<issue>`
- **Verify:** `cd development/ledger && pnpm run verify:tsc` + `pnpm run verify:build` after each chunk

### Loki (Validation)

**Responsibility:** Critique and cull bloat. Validate that tests follow the pyramid.

- **Run bloat critique:** `bash quality/scripts/loki-critique.sh` after every coverage run
- **Review findings:** Run `bash quality/scripts/loki-critique.sh` and check generated output for flagged files
- **Act on CRITICAL:** Files >15 tests = immediate consolidation issue filed
- **Act on WARNINGS:** 3+ warnings in one suite = file consolidation issue
- **Validate handoff:** Ensure all tests pass, tsc green, build green, before approving PR
- **Final QA:** PASS verdict includes new tests written and verifying acceptance criteria

## QA Standards

**Every test must:**
1. Derive assertions from acceptance criteria, not from what the code currently does
2. Be idempotent — clear relevant localStorage before running
3. Map to a user journey or integration point, not to incidental implementation details
4. Avoid CSS measurements, static text assertions, and over-specification

**Every PASS verdict requires:**
- Code review clean
- Build clean (`npm run build`)
- TypeScript clean (`cd development/ledger && pnpm run verify:tsc`)
- GitHub Actions green
- New Playwright tests written and passing (Loki-approved)
