# n8n-workflow — Import & Export n8n Workflows

Manages n8n workflows between the local repo (`infrastructure/n8n/workflows/`) and the
live n8n pod in `fenrir-marketing` namespace on GKE.

## Usage

```
/n8n-workflow import [workflow-file.json]   # Push local JSON → live n8n pod
/n8n-workflow export                        # Pull live n8n pod → local JSON, commit & PR
```

## Common Setup

### Find the n8n pod

Both operations start by locating the pod:

```bash
N8N_POD=$(kubectl get pods -n fenrir-marketing -l app.kubernetes.io/name=n8n,app.kubernetes.io/component=main -o jsonpath='{.items[0].metadata.name}')
echo "Pod: $N8N_POD"
```

If no pod is found, stop and report:
> "No n8n pod found in fenrir-marketing namespace. Is n8n running?"

---

## import — Push local workflows to the live pod

### 1. Determine which workflows to import

- If `$ARGUMENTS` specifies a filename (e.g., `import gmail-reddit-monitor.json`), import only that file.
- If just `import` with no file, import ALL `.json` files from `infrastructure/n8n/workflows/`.
- Validate each file exists. If missing, stop and list available files.

### 2. Copy and import

```bash
kubectl exec -n fenrir-marketing "$N8N_POD" -- mkdir -p /tmp/n8n-import

# For each file:
kubectl cp "infrastructure/n8n/workflows/<file>.json" "fenrir-marketing/$N8N_POD:/tmp/n8n-import/<file>.json"

# Single file:
kubectl exec -n fenrir-marketing "$N8N_POD" -- n8n import:workflow --input=/tmp/n8n-import/<file>.json

# All files:
kubectl exec -n fenrir-marketing "$N8N_POD" -- n8n import:workflow --input=/tmp/n8n-import/

# Clean up:
kubectl exec -n fenrir-marketing "$N8N_POD" -- rm -rf /tmp/n8n-import
```

### 3. Report

```
## n8n Workflow Import Complete

- **Pod:** <N8N_POD>
- **Workflows imported:** <list>
- **Source:** infrastructure/n8n/workflows/

Workflows are now available in the n8n UI. You may need to activate them manually.
```

## export — Pull live workflows to local repo, commit & PR

### 1. Export from pod

```bash
kubectl exec -n fenrir-marketing "$N8N_POD" -- n8n export:workflow --all --output=/tmp/n8n-export.json
kubectl cp "fenrir-marketing/$N8N_POD:/tmp/n8n-export.json" /tmp/n8n-export.json
```

### 2. Split into individual files

```bash
COUNT=$(jq 'length' /tmp/n8n-export.json)
echo "Exported $COUNT workflows"

for i in $(seq 0 $((COUNT - 1))); do
  ID=$(jq -r ".[$i].id" /tmp/n8n-export.json)
  jq ".[$i]" /tmp/n8n-export.json > "infrastructure/n8n/workflows/${ID}.json"
  echo "  -> ${ID}.json"
done
```

### 3. Check for changes

```bash
git diff --stat infrastructure/n8n/workflows/
git status --short infrastructure/n8n/workflows/
```

**If no changes:** report "No workflow changes detected" and stop.

### 4. Branch, commit, push, PR

```bash
DATE=$(date +%Y-%m-%d)
BRANCH="chore/n8n-export-$DATE"

git checkout -b "$BRANCH"
git add infrastructure/n8n/workflows/
git commit -m "chore: export n8n workflows $DATE

Co-Authored-By: Claude Opus 4.6 (1M context) <noreply@anthropic.com>"
git push origin "$BRANCH"

gh pr create \
  --title "chore: export n8n workflows $DATE" \
  --body "$(cat <<'EOF'
## Summary
- Export n8n workflows from fenrir-marketing pod to infrastructure/n8n/workflows/
- Generated by `/n8n-workflow export`
EOF
)" --base main
```

### 5. Report

```
## n8n Workflow Export Complete

- **Pod:** <N8N_POD>
- **Workflows exported to:** infrastructure/n8n/workflows/
- **Branch:** <BRANCH>
- **PR:** <PR_URL>
```

## Notes

- **Namespace:** `fenrir-marketing`
- **Requires:** kubectl configured for GKE, gh CLI authenticated
- **Upsert on import:** Existing workflows (same ID) are overwritten. New IDs create new workflows.
- **Inactive on import:** Imported workflows are inactive by default — activate manually in the n8n UI.
- **Always use absolute paths** in bash commands since agent threads reset cwd.
- **IF node condition IDs must be UUIDs** — n8n v2 IF nodes silently drop conditions with non-UUID IDs on import.
