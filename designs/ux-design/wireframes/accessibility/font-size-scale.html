Font Size Scale: Before vs After

Issue #149 -- Increase base font sizes for readability

This wireframe demonstrates the typography hierarchy at current (BEFORE) and proposed (AFTER) sizes. No theme styling is applied -- structure only. The goal is a proportional scale-up that preserves visual hierarchy while matching modern readability standards (Stripe docs as reference).

Mobile Considerations (375px min)

At 375px, the increased font sizes must not cause horizontal overflow or line wrapping that breaks layout. Key constraints:

  • Card tile stat labels (3-col grid): at 12px (up from 10px), "Credit" / "Fee" / "Bonus" still fit in 3 columns at 375px - 32px padding = 343px usable / 3 = ~114px per column. Safe.
  • Badge text at 12px (up from 10px): "MUSPELHEIM" is the longest realm name at ~11 chars. At 12px uppercase, this is ~90px. Fits in badge with padding.
  • Form inputs at 16px: already the iOS-safe size. No change needed on mobile.
  • H1 at 32px: on mobile, consider scaling down to 28px via responsive class (text-[28px] md:text-3xl). The page title "The Ledger of Fates" at 32px is ~320px wide -- tight at 375px with padding.
  • Sidebar nav at 14px: sidebar is overlay drawer on mobile, 272px wide. 14px nav items fit comfortably.

Responsive recommendation: H1 can use text-[28px] on mobile, text-3xl (32px) on md+ breakpoint. All other sizes are safe at all viewports.

Tailwind Class Migration Map

This section maps the Tailwind class changes needed. The primary strategy is a systematic find-and-replace of text-xs to text-sm, and text-sm to text-base, with exceptions noted below.

Current Class New Class Context / Exceptions
text-xs (12px) text-sm (14px) Default migration for labels, buttons, nav items, body-small
text-sm (14px) text-base (16px) Body text, card issuer names, descriptions
text-base (16px) text-lg (18px) Only where text-base was used for subheadings
text-lg (18px) text-xl (20px) or text-2xl (24px) Section headings -- context dependent
text-2xl (24px) text-3xl (30px) or text-[32px] Page titles (H1)
text-[10px] text-xs (12px) Badges, uppercase labels
text-[11px] text-[13px] Metadata, timestamps

EXCEPTION: Do not blindly find-replace. Some text-xs/text-sm usages are intentional for dense data displays (e.g., LCARS overlay easter egg, Gleipnir fragment modals). Review each file. The 260 occurrences across 51 files should be reviewed component by component.

globals.css change: Update `body { font-size: 14px }` in the base layer to `body { font-size: 16px }`. This sets the new base for any element that inherits font-size from body without an explicit Tailwind class.

Exceptions: Do Not Change

These components use intentionally small text for stylistic or functional reasons. They should be left at their current sizes.

Component Current Size Reason to Preserve
ConsoleSignature.tsx (console ASCII art) Various ASCII art alignment depends on fixed character widths
LcarsOverlay.tsx (easter egg) text-xs, text-[10px] LCARS aesthetic requires compact data-dense text
Gleipnir copyright tooltip (globals.css) 10px Intentionally tiny -- easter egg hint, not primary UI
.note class (wireframes only) 11px Wireframe annotations, not shipped UI
