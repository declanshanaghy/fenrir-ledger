Story 4.5 — Wolf's Hunger Meter + About Modal Completeness

Wireframe covers: the About modal with Wolf's Hunger section added, the ForgeMasterEgg overlay with the same meter, display format variants, data source (signUpBonus.met === true), and mobile full-sheet behavior. No theme colors applied — structure only.

Data Source — SignUpBonus.met Field
Field Type Inclusion Rule Note
signUpBonus.met boolean Only count bonuses where met === true Field name is met (NOT bonusMet) — confirmed Sprint 4 groom
signUpBonus.amount number Sum by type when met === true Points count, miles count, or cashback in cents
signUpBonus.type BonusType Group by: "points" | "miles" | "cashback" Aggregate totals per type, display all in one line
Card scope All cards Active cards + closed (Valhalla) cards — both included The wolf remembers every meal

getCards(householdId) = active cards. getClosedCards(householdId) = Valhalla cards. Both calls required. Filter by signUpBonus?.met === true. Aggregate by type.

Wolf's Hunger Meter — Display Format Variants
Variant 1: All cash back
Fenrir has consumed: $1,240 cashback

Currency: $ prefix, integer (no decimals). toLocaleString() for commas.

Variant 2: All points
Fenrir has consumed: 847,000 pts

Bare number + type label. No $ prefix.

Variant 3: Mixed types (points + miles + cashback)
Fenrir has consumed: 412,000 pts · 95,000 miles · $320 cashback

Separator: " · " (space·space). All types on one line. JetBrains Mono for numbers.

Variant 4: No bonuses met (empty state)
The wolf has not yet fed. Add reward details to your cards.

No "Fenrir has consumed:" label. No "The chain grows heavier." subline. Just the fallback copy.

Atmospheric subline "The chain grows heavier." appears ONLY when any rewards are shown. Format numbers: amount.toLocaleString(). Cash: $N. Points/miles: N type.

Aggregation Logic — State Diagram

Mermaid flowchart — follows ux/ux-assets/mermaid-style-guide.md conventions. 

```mermaid
graph TD
    classDef primary fill:#03A9F4,stroke:#0288D1,color:#FFF
    classDef healthy fill:#4CAF50,stroke:#388E3C,color:#FFF
    classDef neutral fill:#9E9E9E,stroke:#757575,color:#FFF

    modalOpen([Modal opens]) --> fetchActive[getCards(householdId)\nactive cards]
    modalOpen --> fetchClosed[getClosedCards(householdId)\nValhalla cards]
    fetchActive --> combine[Combine all cards]
    fetchClosed --> combine
    combine --> filter{signUpBonus?.met\n=== true?}
    filter -->|No| skip[Skip card\n(0 contribution)]
    filter -->|Yes| aggregate[Sum amount\nby type]
    aggregate --> group{Any totals > 0?}
    group -->|No| fallback[Render fallback copy\nThe wolf has not yet fed...]
    group -->|Yes| render[Render hunger line\nFenrir has consumed: ...]
    render --> subline[Render subline\nThe chain grows heavier.]

    class modalOpen primary
    class render healthy
    class fallback neutral
```
