Touch Target Specifications

Minimum Touch Target Size: 44×44px

All interactive elements must have a touch target of at least 44×44 pixels. This can be achieved through padding, invisible extensions, or parent containers.

Implementation Notes

Form Inputs:
• Set min-height: 44px on all inputs, selects, and textareas
• Use padding: 0.75rem (12px) for comfortable touch spacing
• Font-size: 16px (1rem) prevents iOS Safari zoom on focus
• Border: 2px provides clear visual boundaries
Step Dots:
• Visual dot is only 8×8px but has invisible 44×44px touch target
• Use ::after pseudo-element or wrapper to extend clickable area
• Ensure keyboard navigation with tabindex and role="tab"
Buttons:
• Min-height: 44px with padding: 0.75rem 1rem
• Stack vertically on mobile when horizontal space is limited
• Primary action (Save) should be visually prominent
Responsive Behavior:
• At 375px: Single column layout, stacked buttons
• At 600px+: Can start using 2-column grids for related fields
• At 768px+: Horizontal button groups, side-by-side fieldsets
• Never require horizontal scrolling on mobile
