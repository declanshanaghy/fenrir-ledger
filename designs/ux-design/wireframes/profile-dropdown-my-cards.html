Interaction Spec — My Cards Nav Entry

Behaviour Spec Implementation Note
Navigation target Links to /ledger Use Next.js <Link href="/ledger"> wrapped in a button (same pattern as Settings row). Call onClose() before push.
Position in dropdown Immediately below the profile header (user info), above Theme row Insert before the Theme menuitem div in ProfileDropdown in LedgerTopBar.tsx
Active state When pathname === "/ledger": label is bold, a visual indicator (dot or left border accent) is shown — matching the existing isSettingsActive pattern Add const isMyCardsActive = pathname === "/ledger" alongside existing isSettingsActive. Apply same gold colour class as Settings active state.
Icon Grid / cards icon from lucide-react (e.g. LayoutGrid or CreditCard) Match size and style of Settings icon: className="h-4 w-4 shrink-0"
Touch target min-height: 44px desktop / 48px mobile Existing rows use style={{ minHeight: 44 }} — keep consistent
Keyboard nav Tab order: My Cards → Theme → Settings → Sign out. Activates on Enter/Space. Inherits from role="menuitem" on <button> element — no extra work needed
Dropdown close behaviour Clicking My Cards closes the dropdown then navigates Call onClose() then router.push("/ledger")
Accessibility label Visible text "My Cards" is sufficient. No aria-label needed. Inherits role="menuitem" from parent menu pattern

The "My Cards" entry is only shown in the signed-in ProfileDropdown (not in the anonymous UpsellPromptPanel). Anonymous users have no card list to navigate to.

No change to the anonymous upsell panel (UpsellPromptPanel).

Active state colour token: use text-gold matching the existing isSettingsActive class pattern already in LedgerTopBar.tsx.
