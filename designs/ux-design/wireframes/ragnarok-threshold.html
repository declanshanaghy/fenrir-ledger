Section 2 — Ragnarok Threshold Logic State transition: normal mode → Ragnarok mode → auto-resolves

Threshold Logic: When Ragnarok Activates

Ragnarok mode is a read-only UI state overlay. It produces no alerts, modals, or user-blocking elements. It resolves automatically — no user action required.

Normal Mode
< 3 cards in warning state
Page title: "CARDS" Howl panel: "THE HOWL" (standard) Background: standard dark theme Gold accents: standard gold Status rings: normal / fee pulse Summary bar: no Ragnarok badge
→ ≥ 3 warning
Ragnarok Mode [Easter Egg #8]
≥ 3 cards with fee_approaching or overdue
Page title: "RAGNAROK — FENRIR LEDGER" Howl panel header: altered text (see below) Background: deep red radial overlay (opacity 0.3, z: 9653) Gold accents → blood orange (theme var swap) Warning card rings: pulse faster Summary bar: [RAGNAROK MODE] badge Konami code: double intensity howl
→ < 3 warning
Normal Mode (restored)
Auto-resolves — no user action
Page title reverts to "CARDS" Howl panel header reverts to standard Red overlay fades out (CSS transition) Gold accents restore to standard gold No toast or confirmation shown
Auto-Dismiss Behavior Ragnarok mode resolves automatically when the count of cards with fee_approaching or overdue status drops below 3. This happens when: a card is closed, a fee is paid and status updates, or a card's deadline passes into a non-warning state. No explicit dismiss action exists — Ragnarok ends when the chains loosen.
Konami Code Variant (Easter Egg #2 interaction) Triggering the Konami code (↑ ↑ ↓ ↓ ← → ← → B A) while Ragnarok mode is active plays the wolf howl at double intensity. Normal Konami: standard howl amplitude. Ragnarok Konami: doubled amplitude (AudioContext gain node ×2, clamped at browser limit). Only fires if AudioContext is already running — no autoplay. Wolf-rise animation and FENRIR AWAKENS status text still play as normal.
