Interaction Specification -- Stripe Upsell Banner Lifecycle

Render condition

subscriptionTier === 'thrall' AND localStorage.getItem('fenrir:stripe_upsell_dismissed') !== 'true'
Evaluated on every dashboard page load. Not shown on /valhalla, /cards/*, /settings.

CTA flow

  1. User clicks "Upgrade to Karl".
  2. If anonymous: open email collection modal (see anonymous-checkout.html).
  3. If authenticated: proceed directly.
  4. POST /api/stripe/checkout-session with email + priceId.
  5. Redirect to Stripe Checkout URL.
  6. On success: user returns via success_url with Karl tier active.
  7. Banner dismiss flag is NOT set by the CTA -- but the banner will not render because tier is now Karl.

Dismiss flow

  1. User clicks x dismiss button.
  2. Immediately: localStorage.setItem('fenrir:stripe_upsell_dismissed', 'true').
  3. Collapse animation: height to 0, opacity to 0, 300ms ease.
  4. After animation: element removed from DOM.
  5. On all future loads: banner is not rendered.

Re-entry point

Users who dismissed the banner can still subscribe from /settings (StripeSettings component). The SealedRuneModal also provides a subscribe CTA when accessing locked features.
