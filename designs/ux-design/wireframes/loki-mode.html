[RANDOM REALM] placeholders cycle rapidly through realm names (Midgard / Asgard / Helheim / Niflheim / etc.) — a "shapeshifter flash" effect. Actual card data is never modified; only display order and badge labels are temporarily overridden in component state. Restoration fades back with a CSS transition matching saga-enter easing.

Toast Notification (z-index: 300)

Simulated viewport top-right corner. Toast appears immediately when Loki Mode activates. Dismissed automatically when order restores (5 s), or on click.

Uses the same toast component as card-count milestone toasts (Easter Egg #11). No custom toast layout is required — copy text is the only Loki-specific override. Entry animation: slide-in from right edge (matches existing toast convention).

Fenrir Ledger · Forged by FiremanDecko · Designed by Luna · Owned by Freya · Tested by Loki [ 0 / 7 clicks ]

"Loki" text — Click 7 times to trigger. No visual affordance in normal state. Click counter resets after Loki Mode completes its 5-second cycle. Counter is held in component state only (not persisted to localStorage). The loki-click-counter element shown here is a wireframe annotation label only — it does NOT render in the production UI.

Interaction State Flow

State machine (see designs/ux-design/interactions.md or sprint doc for rendered Mermaid): 

stateDiagram-v2
    [*] --> Normal : page loads

    Normal --> Counting : user clicks "Loki" in footer
    Counting --> Counting : click < 7 (counter increments)
    Counting --> Normal : 30 s idle — counter resets

    Counting --> LokiActive : click 7 reached
    LokiActive --> LokiActive : card grid scrambled\nstatus badges cycling random realms\ntoast displayed\ntimer running

    LokiActive --> Restoring : 5 s elapsed
    Restoring --> Normal : fade-back transition complete\ncounter resets

Render the above in Mermaid stateDiagram-v2 syntax following designs/ux-design/ux-assets/mermaid-style-guide.md conventions.
