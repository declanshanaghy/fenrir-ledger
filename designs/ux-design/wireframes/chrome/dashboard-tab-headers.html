Dashboard Tab Headers, Tooltips, and Empty States

Wireframe for 4 new features inside the Ledger of Fates dashboard tabs: (1) simplified runic empty states, (2) card status label tooltips, (3) dismissable tab header explaining each tab's purpose, (4) dismissable tab summary sub-header showing dynamic card data. Applies to all 5 tabs: ALL, VALHALLA, ACTIVE, THE HUNT, THE HOWL.

Shows all three layers inside a tab panel: dismissable header, dismissable summary sub-header, then card grid. Both header and summary can be independently dismissed.

Scenario 6 — Empty States for All 5 Tabs

Simplified empty states using Elder Futhark rune decoration. Format: rune + short text + rune. No explanatory subtext. Voice 2 (atmospheric). Each tab uses a distinct rune.

ALL TAB EMPTY STATE
ᚅ No cards in the ledger ᚅ

Rune: ᚅ (Othalan). Shown when card array is empty.

VALHALLA TAB EMPTY STATE
ᛘ No cards in Valhalla ᛘ

Rune: ᛘ (Tiwaz). Matches the Valhalla rune from mythology-map.md.

ACTIVE TAB EMPTY STATE
ᚅ No active cards ᚅ

Rune: ᚅ (Othalan). Matches the existing tab rune.

THE HUNT TAB EMPTY STATE
ᛜ No cards on the hunt ᛜ

Rune: ᛜ (Ingwaz). Matches the Hunt tab rune.

THE HOWL TAB EMPTY STATE
ᚲ No cards in the howl ᚲ

Rune: ᚲ (Kenaz). Matches the Howl tab rune.

Pattern: "[rune] No [category noun] [rune]"
Voice: Atmospheric (Voice 2). Runes are decorative, flanking the text.
Previous design: The old empty states had multi-line explanatory subtext (e.g. "All your cards are currently in The Howl. Add a card to see it here."). This is replaced by the simpler runic pattern above. The tab header (when visible) already explains what cards belong in each tab, so the empty state only needs to confirm "nothing here."

Implementation: Use a Tooltip component (e.g. Radix/shadcn Tooltip).
Desktop: Show on hover with 200ms delay. Dismiss on mouse leave.
Mobile: Show on tap. Dismiss on tap outside or after 3s timeout.
Position: Below the badge by default; flip above if near viewport bottom.
ARIA: Badge gets aria-describedby pointing to tooltip content. Tooltip content is in a role="tooltip" element.
Max width: 260px. Line 1 (bold label) + Line 2 (plain meaning) + Line 3 (italic Norse flavor).

Component Spec — TabHeader
Prop Type Description
tabId DashboardTab Which tab this header belongs to. Used to construct the localStorage key.
title string Bold header title (e.g. "The Howl -- Where Chains Tighten").
description string Explanation of the tab's purpose.
statuses string Description of card status labels found in this tab.
State Behavior
Visible (default) Rendered when localStorage.getItem("fenrir:tab-header-dismissed:{tabId}") is not "true".
Dismissed On dismiss click: set localStorage key to "true", remove from DOM (no animation).
Accessibility:
- Dismiss button: aria-label="Dismiss tab guide"
- Header container: no special ARIA role needed (it is informational content, not a landmark)
- On mobile, the statuses line may be hidden (hidden sm:block) to save vertical space
Component Spec — TabSummary
Prop Type Description
tabId DashboardTab Which tab this summary belongs to. Used to construct the localStorage key.
cards Card[] The filtered cards for this tab. Summary text is computed from this array.
Tab Summary Text Pattern
ALL {total} cards total: {active} active, {hunt} hunting, {howl} howling, {valhalla} in Valhalla
VALHALLA {closed} closed, {graduated} graduated -- chains broken, plunder secured
ACTIVE {count} active cards in good standing
THE HUNT {count} cards with open bonus windows, {approaching} approaching deadline
(approaching = bonus deadline within 30 days)
THE HOWL {fee} with fee due, {promo} promo expiring, {overdue} overdue
(omit zero-count segments)
State Behavior
Visible (default) Rendered when localStorage key is not "true" AND cards.length > 0.
Empty tab Not rendered. When the tab has zero cards, there is nothing to summarize.
Dismissed On dismiss click: set fenrir:tab-summary-dismissed:{tabId} to "true".
Dynamic updates: The summary text recomputes whenever cards changes. It always reflects the current state of the portfolio.
Accessibility: Dismiss button: aria-label="Dismiss tab summary".
Component Spec — StatusTooltip
Prop Type Description

children ReactNode The badge/trigger element to wrap.

Implementation notes:
- Wrap the existing status badge in a shadcn/ui Tooltip component.
- The tooltip content has 3 lines: bold label, plain meaning, italic Norse flavor.
- Desktop: delayDuration={200}. Mobile: tap to toggle.
- role="tooltip" on the content, aria-describedby on the trigger.
- Max width: 260px. Z-index: 300 (toast layer — above cards/panels).

Reference — localStorage Keys
Key Value Purpose
fenrir:tab-header-dismissed:all "true" ALL tab header dismissed
fenrir:tab-header-dismissed:valhalla "true" VALHALLA tab header dismissed
fenrir:tab-header-dismissed:active "true" ACTIVE tab header dismissed
fenrir:tab-header-dismissed:hunt "true" THE HUNT tab header dismissed
fenrir:tab-header-dismissed:howl "true" THE HOWL tab header dismissed
fenrir:tab-summary-dismissed:all "true" ALL tab summary dismissed
fenrir:tab-summary-dismissed:valhalla "true" VALHALLA tab summary dismissed
fenrir:tab-summary-dismissed:active "true" ACTIVE tab summary dismissed
fenrir:tab-summary-dismissed:hunt "true" THE HUNT tab summary dismissed
fenrir:tab-summary-dismissed:howl "true" THE HOWL tab summary dismissed
Namespace: All keys use the fenrir: prefix per project convention.
Pattern: fenrir:tab-{header|summary}-dismissed:{tabId}
Clearing: No UI to un-dismiss. Users would need to clear localStorage manually or the app could add a "Reset tips" option in Settings (future consideration).
Accessibility Requirements
Element Requirement
Tab header dismiss button aria-label="Dismiss tab guide". Touch target: min 44x44px (visual size can be smaller with extended padding).
Tab summary dismiss button aria-label="Dismiss tab summary". Same touch target requirement.
Status badge tooltip role="tooltip" on content. Trigger gets aria-describedby. Tooltip dismissible via Escape key.
Empty state runes Runes are decorative. The text between them carries the semantic meaning. No aria-hidden needed since the runes are inline with the text and screen readers will read the full string.
Tab header content No ARIA landmark needed. Content is informational, not a navigation or region landmark.
Focus management on dismiss After dismissing header: focus moves to the first focusable element in the tab panel (summary dismiss button, or first card). After dismissing summary: focus moves to the first card.
