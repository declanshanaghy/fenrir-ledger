Scenario 1 -- Primary Button (Gold) -- All States
btn-primary

Used for: "Subscribe for $3.99/month", "Add Card", "Save changes", "Manage Subscription", "Resubscribe". The highest-emphasis button.

Default
Solid border. Bold label. Full opacity. cursor: pointer.
:hover
CSS-only. Border thickens to 2px. In theme: brightness(1.15) + subtle border glow (gold). Transition: 150ms ease-out.
:active
CSS-only. Scale down to 0.97. In theme: brightness(0.9) + inset shadow feel. Transition: 80ms ease-out. Immediate visual "press" feedback.
Loading
React state. Spinner (14px, border animation, 0.8s linear infinite rotate) + text changes to "Loading...". Button disabled. Opacity: 0.7. cursor: not-allowed. aria-busy="true" + aria-label updated.
Disabled
Static disabled state (not loading -- just unavailable). Opacity: 0.4. cursor: not-allowed. No hover/active effects. aria-disabled="true".

Primary buttons are the highest-emphasis CTA. In the theme: gold background (#c9920a), dark text (#07070d). Hover brightens the gold. Active darkens it and scales down. Loading replaces the label with spinner + "Loading..." and disables interaction.
The 150ms hover and 80ms active durations match the existing interaction spec (see interactions.md -- hover: 150-200ms).

Scenario 2 -- Secondary / Ghost Button -- All States
btn-secondary / btn-ghost

Used for: "Cancel", "Manage Subscription" (in canceled state), "Continue without signing in", "Not now". Outlined or borderless style.

Default
1px solid border. Normal weight. Full opacity.
:hover
CSS-only. Border brightens. In theme: background shifts to subtle fill (rgba white 5%). Transition: 150ms ease-out.
:active
CSS-only. Scale(0.97). In theme: background darkens slightly. Transition: 80ms ease-out.
Loading
Same pattern as primary. Spinner + text + disabled. Used when secondary buttons trigger API calls (e.g., "Manage Subscription" in canceled state opens Portal).
Disabled
Opacity: 0.4. cursor: not-allowed. No hover/active.
Scenario 3 -- Destructive Button -- All States
btn-destructive

Used for: "Cancel" (subscription cancel), "Delete card". Visually de-emphasized but distinct as a danger action.

Default
1px solid border. In theme: muted red/danger color, not prominent. Normal font weight.
:hover
CSS-only. In theme: danger color intensifies. Background gains subtle red tint. Transition: 150ms ease-out.
:active
CSS-only. Scale(0.97). Danger color deepens. Transition: 80ms ease-out.
Loading
Spinner + contextual text ("Canceling...", "Deleting..."). Button disabled. Same opacity/cursor as other variants.
Disabled
Opacity: 0.4. cursor: not-allowed.
Scenario 4 -- Loading Indicator Detail
Loading state anatomy
Loading...

Spinner: 14 x 14px. Circular border with one transparent side. CSS animation: rotate 360deg, 0.8s, linear, infinite.

Text: Replaces the original label. Contextual: "Loading...", "Redirecting...", "Saving...", "Canceling..."

Gap: 8px between spinner and text.

Alignment: Spinner and text vertically centered within button.

Contextual loading text by button:

Button Default Label Loading Label API Call
Manage Subscription Manage Subscription Redirecting... POST /api/stripe/portal
Subscribe Subscribe for $3.99/month Redirecting... POST /api/stripe/checkout-session
Add Card Add Card Saving... localStorage (fast, but keep pattern consistent)
Save Changes Save changes Saving... localStorage
Cancel (subscription) Cancel Redirecting... POST /api/stripe/portal
Delete Card Delete card Deleting... localStorage
Resubscribe Resubscribe Redirecting... POST /api/stripe/checkout-session

For localStorage operations (Add Card, Save, Delete), the loading state may flash too quickly. Use a minimum display time of 300ms to prevent jarring flicker. For API calls (Stripe portal/checkout), show loading until redirect or error. On error, revert to default state and show error toast.

Scenario 6 -- Mobile 375px -- Button States
Default State (375px)
Loading State (375px)

When one button in a group is loading, sibling action buttons are disabled (opacity 0.4) to prevent conflicting actions.

Mobile: all buttons full-width, stacked vertically. Touch targets remain 44px minimum height.
Hover states still apply for devices with hover capability (@media (hover: hover)).
Active/pressed state (scale 0.97) is especially important on mobile -- provides immediate tap feedback on touchscreens where hover is unavailable.

Accessibility Requirements
Element Requirement
All buttons Visible focus ring on keyboard focus (:focus-visible). Must meet WCAG 2.1 AA contrast (3:1 for UI components). Min touch target 44x44px.
Loading state aria-busy="true" on the button while loading. aria-disabled="true" alongside disabled attribute. aria-label updated to include loading context (e.g., "Manage Subscription, loading").
Spinner aria-hidden="true" on the spinner SVG/element -- it is decorative. The loading text ("Redirecting...") is the accessible indicator. Spinner respects prefers-reduced-motion: reduce -- stops rotation, shows static circle.
Hover/active states Do NOT rely on hover alone for critical information. Hover feedback is enhancement, not gating. All information available without hover (keyboard, touch).
Reduced motion @media (prefers-reduced-motion: reduce): disable scale transforms on :active, disable spinner rotation (show static indicator), keep opacity changes (they are not motion).

CSS Implementation Summary

/* All button variants share these transition foundations */
.btn {
    transition:
        transform 80ms ease-out,
        filter 150ms ease-out,
        border-color 150ms ease-out,
        background-color 150ms ease-out,
        box-shadow 150ms ease-out,
        opacity 150ms ease-out;
}

/* Hover -- CSS-only, all variants */
.btn:not(:disabled):hover {
    filter: brightness(1.15);
    /* Primary: border glow gold */
    /* Secondary: subtle background fill */
    /* Destructive: danger color intensifies */
}

/* Active / Pressed -- CSS-only, all variants */
.btn:not(:disabled):active {
    transform: scale(0.97);
    filter: brightness(0.9);
}

/* Loading -- applied via React state (isLoading prop) */
.btn[aria-busy="true"] {
    opacity: 0.7;
    cursor: not-allowed;
    pointer-events: none;
}

/* Disabled -- static unavailable state */
.btn:disabled:not([aria-busy="true"]) {
    opacity: 0.4;
    cursor: not-allowed;
}

/* Spinner rotation */
@keyframes btn-spinner {
    to { transform: rotate(360deg); }
}
.btn-spinner {
    width: 14px;
    height: 14px;
    border: 2px solid currentColor;
    border-top-color: transparent;
    border-radius: 50%;
    animation: btn-spinner 0.8s linear infinite;
}

/* Reduced motion */
@media (prefers-reduced-motion: reduce) {
    .btn:not(:disabled):active { transform: none; }
    .btn-spinner { animation: none; }
}
