Signed-in users never see the upsell banner.
The banner render condition: !isSignedIn AND !dismissed.
Both conditions must be true for the banner to appear.
After sign-out, the user returns to anonymous state — the dismiss flag is re-checked. If they had previously dismissed the banner, it stays dismissed after sign-out.

Interaction Specification — Banner Lifecycle

Render condition

isAnonymous AND localStorage.getItem('fenrir:upsell_dismissed') !== 'true'
Both conditions must be true. Evaluated on every page load on the dashboard route only.

Dismiss flow

  1. User taps × dismiss button.
  2. Immediately: localStorage.setItem('fenrir:upsell_dismissed', 'true').
  3. Collapse animation starts: height → 0px, opacity → 0, 300ms ease.
  4. After animation: element removed from DOM. Grid row 2 collapses. Content moves up.
  5. On all future loads: banner is not rendered at all.

"Sign in to sync" CTA flow

  1. User taps "Sign in to sync".
  2. Navigate to /sign-in — the dedicated sign-in page.
  3. Banner dismiss flag is NOT set — if the user abandons sign-in and returns, the banner is still there.
  4. After successful sign-in: banner disappears (user is no longer anonymous). Dismiss flag irrelevant.
  5. After failed or abandoned sign-in: banner reappears on dashboard (not dismissed).

Settings fallback

The settings page (/settings — "The Ravens") includes a persistent "Sync to cloud" option. This is the permanent re-entry point for users who dismissed the banner and later decide to sign in. See the settings wireframe (future sprint) for the spec.
