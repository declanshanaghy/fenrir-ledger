Interaction Spec

Tab Classification Logic

  • The Howl tab receives cards with status: fee_approaching, promo_expiring, overdue.
  • Active tab receives cards with status: active — i.e. all cards NOT in The Howl.
  • Each card appears in exactly ONE tab. No duplication across tabs.
  • Status is computed by existing card-utils.ts getCardStatus() — no new logic needed for classification.

Default Tab Selection

  • On initial render: if howlCards.length > 0, default to The Howl tab.
  • On initial render: if howlCards.length === 0, default to Active tab.
  • Tab selection is client-side state only (useState). No URL params, no persisted preference.
  • When cards are updated (added/edited/deleted), re-evaluate default tab rule only on fresh mount — do not reset tab mid-session.

Tab Switching Behavior

  • Tab switching is instant — client-side show/hide. No page reload, no network request.
  • Use role="tablist", role="tab", role="tabpanel" ARIA pattern.
  • Active tab: aria-selected="true". Inactive: aria-selected="false".
  • Tab panel visibility: use hidden attribute or conditional render. Either is acceptable — engineer's choice based on animation needs.
  • Keyboard: ← / → arrow keys navigate between tabs (standard ARIA tabs pattern).
  • Transition: no required animation spec for tab switch. Optional: 150ms fade on panel content.

Badge Count Updates

  • Badges reflect live derived counts: howlCards.length and activeCards.length.
  • Badges update automatically when cards array changes (derive from prop/state, no separate badge state).
  • When Howl count drops to 0: badge shows 0 at reduced opacity (or hide badge entirely — engineer's preference). Do NOT remove the tab button itself.
  • The Howl badge uses a visually distinct style (heavier border) to signal urgency when count > 0.

Howl Tab Urgency Styling

  • Cards in The Howl tab display an urgency bar above the card header (see .card-urgency-bar).
  • Urgency bar content: indicator dot + status label + days remaining (or "X days past" for overdue).
  • Urgency dot: pulsing animation for overdue cards; static for fee_approaching and promo_expiring.
  • Sort order within The Howl: ascending by days remaining — most urgent (lowest/negative days) at top.
  • Reuse existing Howl panel urgency styling tokens from HowlPanel.tsx and theme system.
  • The ᚲ Kenaz rune badge on The Howl tab should pulse when count > 0 (matches existing HowlPanel header behavior).

Mobile Tab Navigation

  • Tab bar is horizontally scrollable (overflow-x: auto, -webkit-overflow-scrolling: touch, scrollbar hidden).
  • Minimum viewport: 375px. All tab text and badges must be readable at 375px with default system font size.
  • Touch target for each tab: minimum 44px height (accessible tap area).
  • The Howl bottom drawer (current pattern) is removed — tabs are the sole navigation mechanism on mobile.
  • The bell/raven icon in the mobile TopBar is also removed — no longer needed once tabs replace the drawer.

Layout Changes from Current Dashboard

  • Removed: Right-side AnimatedHowlPanel / HowlPanel sidebar on desktop.
  • Removed: Mobile bottom drawer for Howl panel.
  • Removed: mobileHowlOpen state in page.tsx.
  • Added: Tab bar below content header, spanning full content column width.
  • Added: activeTab state: 'howl' | 'active'.
  • Changed: Card grid now renders only the subset matching the active tab.
  • Sidebar, TopBar, footer, content header: unchanged.

Component Reuse Notes (for FiremanDecko)

  • Reuse card-utils.ts — no new status computation. Classification is purely: ['fee_approaching','promo_expiring','overdue'].includes(card.status).
  • The Howl tab card urgency bar can be a thin wrapper/variant prop on the existing CardChain component.
  • Consider shadcn/ui Tabs (Tabs, TabsList, TabsTrigger, TabsContent) for accessible tab implementation.
  • The HowlPanel.tsx urgency row logic (toUrgentRows()) can be adapted for the in-tab card urgency bar.
  • The Howl tab empty state reuses the existing ᚱ Raido rune + "The wolf is silent" copy from HowlPanel.
