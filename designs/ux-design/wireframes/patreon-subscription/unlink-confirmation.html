Scenario 1: Desktop — Unlink Patreon Confirmation

Triggered when user clicks "Unlink Patreon" on the Settings page. All copy is Voice 1 (functional). No Norse language -- this is a consequential action confirmation that must be unambiguous. The dialog explains exactly what happens and what does not happen.

Backdrop (semi-transparent overlay, z-index: 200)

Unlink Patreon?

Your Patreon account will be disconnected from Fenrir Ledger. You will lose access to premium features, but your card data will not be affected.

If you have an active Patreon membership, it will continue on Patreon until you cancel it there.

Scenario 2: Mobile 375px — Unlink Confirmation

Mobile layout. Buttons stack vertically per mobile form convention: "Unlink Patreon" on top (primary/destructive), "Cancel" below. Both buttons are full-width with min-height 48px for touch targets. Dialog is nearly full-width (92vw = ~345px on 375px viewport).

Backdrop (375px viewport)

Unlink Patreon?

Post-Unlink Behavior Notes

After confirming unlink: 1. Patreon link is removed from user profile (localStorage + server cache). 2. Tier badge disappears from TopBar immediately. 3. Premium features become locked (hard gate modal on next attempt). 4. Toast appears: "Patreon unlinked." (Voice 1, polite aria-live). 5. Settings Patreon section returns to "Link Patreon" button state (see settings-patreon-unlinked.html). 6. No page navigation -- user stays on Settings page. 7. Focus moves to the Patreon section heading after the dialog closes.
