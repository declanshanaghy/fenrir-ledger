The Watcher Speaks
Executive Summary

Heimdall's gaze swept across the ramparts of fenrirledger.com and the hidden tower of Odin's Throne on 2026-03-23. The Ledger wall stands largely firm — nineteen of twenty-one API gates are properly sealed, transport encryption is strong, and the main application's security posture has improved markedly since the March 2026 audits. However, a breach of critical severity has been discovered in the watchtower itself.

The Odin's Throne monitor UI at odins-throne.fenrirledger.com is fully accessible to the unauthenticated public. An oauth2-proxy sidecar meant to guard this internal interface has been misconfigured with --skip-auth-regex=.*, a flag that instructs the proxy to skip authentication for every route without exception. Any visitor on the open internet may read all active and completed GKE agent job metadata, session IDs, branch names, PR titles, internal namespace names, and base64-encoded task prompts. This constitutes an unauthorized window into the realm's internal operations and must be sealed immediately.

Beyond the watchtower breach, this audit also confirms that several findings from the March 2026 pen tests remain open: an unpatched SheetJS library (HIGH), unsafe-inline in CSP (MEDIUM), and rate-limiting gaps on key endpoints (MEDIUM). These are documented herein with remediation paths. All new findings discovered in this audit are summarized below.

Immediate Action Required The Odin's Throne oauth2-proxy misconfiguration is externally confirmed. Remove --skip-auth-regex=.* from the oauth2-proxy deployment and redeploy immediately. This is the only CRITICAL finding from this audit.

Detailed Findings — New (This Audit)
Critical PT2026-CRIT-001 Odin's Throne: oauth2-proxy Auth Bypass via --skip-auth-regex=.*
Category
A01 Broken Access Control / Authentication Bypass
Target
odins-throne.fenrirledger.com
File
infrastructure/helm/odin-throne/templates/deployment.yaml line 78
CVSS
9.1 (AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:N/A:N) — estimated
Description
The Odin's Throne monitor is deployed behind an oauth2-proxy sidecar container configured with the argument --skip-auth-regex=.*. This flag instructs the proxy to skip authentication validation for any request path matching the provided regex. The pattern .* matches every path, rendering the proxy's authentication gate completely inert. All requests — regardless of whether they carry a valid session cookie — are forwarded directly to the upstream Hono server without any identity check.
Impact
Any unauthenticated actor on the public internet can access the full Odin's Throne UI and API without credentials. The exposed surface includes: all GKE agent job names and session IDs, job start/completion times and status, branch names and PR/issue titles from job annotations, base64-encoded task prompts (TASK_PROMPT env var) embedded in job kubectl.kubernetes.io/last-applied-configuration annotations, internal K8s namespace names (fenrir-agents), and live agent log WebSocket streams. At time of test, 7 agent jobs were enumerated, including 1 active job.
Status
OPEN — confirmed externally on 2026-03-23
Evidence — oauth2-proxy misconfiguration
# infrastructure/helm/odin-throne/templates/deployment.yaml, lines 75-79
args:
  - --skip-auth-route=GET=^/healthz$   # intentional: K8s health probe exemption
  - --skip-auth-regex=.*               # BUG: skips auth for ALL routes
  - --cookie-secure=true
Evidence — unauthenticated access confirmed via curl
$ curl -sI https://odins-throne.fenrirledger.com/
HTTP/2 200
content-type: text/html; charset=utf-8
# No oauth2-proxy redirect, no authentication challenge — direct 200

$ curl -s https://odins-throne.fenrirledger.com/api/jobs | head -1
{"jobs":[{"name":"agent-issue-1874-step1-luna-501975d4","namespace":"fenrir-agents",
 "status":"succeeded",...,"annotations":{"fenrir/branch":"ux/issue-1874-light-theme-contrast",
 "TASK_PROMPT":"WW91IGFyZSBMdW5hLCB0aGUgVVgg..."}},...]}
# 7 jobs exposed — 1 active, session IDs, branches, encoded prompts all visible
Forging the Fix
# Remove the --skip-auth-regex=.* line entirely.
# The --skip-auth-route=GET=^/healthz$ line must be preserved for K8s probes.
# After removing the offending arg, redeploy:
helm upgrade odin-throne ./infrastructure/helm/odin-throne \
  -f ./infrastructure/helm/odin-throne/values-prod.yaml \
  --set app.image.tag=$IMAGE_TAG

# Verify auth is enforced post-deploy:
curl -sI https://odins-throne.fenrirledger.com/
# Expected: HTTP/2 302 → Location: /oauth2/start?rd=...

Low PT2026-LOW-002 Build Commit Hash Exposed in /api/health Response
Category
A05 Security Misconfiguration / Version Disclosure
Target
https://www.fenrirledger.com/api/health
Description
The health endpoint returns version and buildId fields populated from NEXT_PUBLIC_APP_VERSION and NEXT_PUBLIC_BUILD_ID. At time of test these contained the full git commit SHA of the deployed build.
Impact
Low. Allows attackers to precisely identify the deployed commit, cross-reference against public git history for vulnerability research, and confirm whether security patches have been applied. The endpoint is intentionally public (required for K8s liveness/readiness probes).
Status
OPEN — new finding
Evidence
$ curl -s https://www.fenrirledger.com/api/health
{"status":"ok","timestamp":"2026-03-23T18:47:18.185Z",
 "version":"d4d91c9f6bb7d90b0b9a2f71dab3f1b2e8374b9a",
 "buildId":"d4d91c9f6bb7d90b0b9a2f71dab3f1b2e8374b9a"}
Forging the Fix
// Option A: Return only timestamp (preferred — K8s only needs 200 OK)
return NextResponse.json({ status: "ok", timestamp: new Date().toISOString() });

// Option B: Return a short, opaque build token rather than full commit SHA
// e.g., first 8 chars only, or a build date YYYY-MM-DD

Info PT2026-INFO-002 Naked Domain Redirect Includes Explicit Port 443
Category
Configuration / Info
Description
HTTP requests to fenrirledger.com (naked domain) redirect to https://www.fenrirledger.com:443/ — the port 443 is explicitly included in the Location header. While RFC 2616 allows this, explicitly including the default port is non-standard and may cause issues with strict URL validators or proxy chains.
Status
INFO — no security impact, cosmetic improvement
Evidence

Forging the Fix
Remediation Recommendations — Prioritized
  • Critical
    Remove --skip-auth-regex=.* from Odin's Throne oauth2-proxy (PT2026-CRIT-001) Delete line 78 from infrastructure/helm/odin-throne/templates/deployment.yaml and run helm upgrade. Verify with a curl check that unauthenticated requests to odins-throne.fenrirledger.com receive a 302 redirect to the OAuth2 login page rather than a 200 with data. File a GitHub issue and assign to FiremanDecko for immediate resolution.
  • High
    Patch or replace SheetJS (HIGH-001 carry-forward, issue #476) GHSA-4r6h (prototype pollution) and GHSA-5pgg (ReDoS) affect the current SheetJS version. Upgrade to xlsx v0.20.x or migrate to an alternative (e.g., exceljs). This has been open since the March 10 pentest.
  • Medium
    Resolve CSP unsafe-inline (MEDIUM-001 carry-forward, issue #477) Restore nonce-based CSP or adopt a strict hash-based policy compatible with Next.js RSC streaming. The nonce-based approach attempted previously was reverted due to CDN caching incompatibility — consider serving CSP headers only via middleware (already in place) and disabling edge caching for HTML responses.
  • Medium
    Add rate limiting to /api/auth/token (SEV-003 carry-forward) The PKCE token exchange endpoint has no rate limiting. Implement IP-based rate limiting (e.g., 10 requests/minute per IP) using the existing rateLimit() utility already used in /api/trial/status.
  • Medium
    Fix SSRF in CSV fetch / validate URL before following redirects (MEDIUM-002 carry-forward, issue #478) The CSV import pipeline follows HTTP redirects without validating that the final destination is a safe external URL. Add an allowlist or SSRF validator that blocks private IP ranges (10.x, 172.16.x, 192.168.x, 169.254.x) before each fetch in the redirect chain.
  • Low
    Suppress x-powered-by header (PT2026-LOW-001) Add poweredByHeader: false to next.config.ts. One-line fix, zero functional impact.
  • Low
    Redact commit SHA from /api/health (PT2026-LOW-002) Return only a short build token or date rather than the full git SHA. K8s probes only require a 200 status code — the body content is unused by Kubernetes itself.
  • Info
    Add /.well-known/security.txt (PT2026-INFO-001) Create public/security.txt with a contact email, expiry date, and canonical URL. This enables responsible disclosure and signals to security researchers that reports are welcome.
