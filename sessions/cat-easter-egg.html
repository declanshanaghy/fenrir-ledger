Act II · Investigation

The Hunt for the Indicator

FiremanDecko

Inspec the main page and find the silent background sync indicator

The wolf searched every corner of the rendered app: page.tsx, Dashboard.tsx, TopBar.tsx, AppShell.tsx, SideNav.tsx. Nothing. The silent background sync indicator described in design/easter-eggs.md as a "tooltip on the silent background sync indicator (bottom-right corner)" does not exist in the codebase — it was a planned element that had never been built.

The easter-eggs design doc places Gleipnir Fragment 1's trigger there. The indicator must be created before the egg can be wired. The proposal: a tiny dim dot, fixed bottom-right, pulsing gold when syncing. cursor-default — no affordance. A tooltip reveals the fragment text on hover. A click triggers the egg.

Design Decision

Click (not hover) triggers the egg — browser autoplay policy requires new Audio().play() to be called inside a direct user-gesture handler. Same pattern as the Women's Beard button in the About modal.

Act III · Investigation

The False Sign Unmasked

FiremanDecko

What is this element in the page? <div role="status" class="nextjs-toast nextjs-static-indicator-toast-wrapper">...Static route...</div>

The element is Next.js's built-in Static Route Indicator — a dev-mode overlay injected automatically by the framework showing whether the current route is statically or dynamically rendered. The lightning bolt is the Next.js logo. It appears only during next dev and is stripped in production builds.

FiremanDecko also corrected the wolf's assumption: this indicator appears bottom-left, not bottom-right. The bottom-right corner is clear. The real sync indicator can live there in production with no conflict.

// Disable Next.js dev indicator in next.config.ts if needed:
const nextConfig = {
  devIndicators: {
    appIsrStatus: false,  // hides the Static/Dynamic indicator
  },
};

Act IV · New Feature

The Silent Pulse Is Born 🔊

Three files touched in one pass. SyncIndicator.tsx is the new component — a fixed bottom-right dot that idles as near-invisible iron (#2a2d45) and pulses gold (#c9920a) with a animate-ping ring when syncing. A CSS group-hover tooltip (no Radix dep needed) floats above the dot on hover, revealing the fragment text in gold mono: The sound of a cat's footfall. Clicking the dot — cursor-default, no visible affordance — fires useGleipnirFragment1() if not yet discovered.

Sync pulses are real and fake. storage.ts now dispatches a fenrir:sync CustomEvent after every setAllCards() write. SyncIndicator listens for it — so saving or deleting a card triggers a genuine pulse. A setInterval at 45-second intervals provides the ambient fake cadence, and an immediate pulse fires on mount so the animation is visible from the first page load.

// storage.ts — real sync event on every card write
localStorage.setItem(STORAGE_KEYS.CARDS, JSON.stringify(cards));
window.dispatchEvent(new CustomEvent("fenrir:sync"));

// SyncIndicator.tsx — listens for real + fires fake cadence
useEffect(() => {
  pulse(); // immediate on mount
  const id = setInterval(pulse, 45_000);
  window.addEventListener("fenrir:sync", pulse);
  return () => { clearInterval(id); window.removeEventListener("fenrir:sync", pulse); };
}, [pulse]);
  • Dot idle colour #1e2235 (rune-border) → #2a2d45 (iron-border) — slightly more visible
  • Tooltip: CSS group-hover no Radix @radix-ui/react-tooltip dep required
  • Audio: triggered inside onClick handler browser autoplay policy satisfied
