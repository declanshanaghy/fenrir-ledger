⏺ That's Next.js's built-in Static Route Indicator — a dev-mode overlay that
  Next.js injects automatically in the bottom-right corner to show whether the
  current route is statically or dynamically rendered. The lightning bolt icon
  is Next.js's logo.

  It's not part of the app's own code — it only appears during next dev and is
  stripped in production builds. You can dismiss it with the X button, or
  disable it permanently in next.config.ts:

  const nextConfig = {
    devIndicators: {
      appIsrStatus: false,   // hides the Static/Dynamic indicator
    },
  };

⏺ Here's the interaction spec before a single line of code:

  ---
  Silent Sync Indicator — Interaction Definition

  Visual

  - Fixed bottom-4 right-4, z-50
  - A single small dot (8px) + optional micro-label "sync" in font-mono 
  text-[9px]
  - Idle: dot bg-[#1e2235] (nearly invisible against dark bg), no motion
  - Syncing: dot pulses bg-[#c9920a], ring expands outward via animate-ping
  clone — gold ripple, ~1.5s, then settles back to idle
  - cursor-default — no pointer affordance, wolf rewards curiosity

  Fake Sync Activity

  - On mount: fire an immediate sync pulse (1.5s) so it's visible on load
  - Then every 45s: another 1.5s pulse
  - Also fires a pulse whenever a card is saved (we can emit a custom event from
   storage.ts)

  Easter Egg Interaction

  - Hover: Radix <Tooltip> appears — content is the raw fragment text in mono:
  The sound of a cat's footfall
  - No label, no "easter egg" hint. Just the text, slightly gold.
  - Click anywhere on the indicator: if egg:gleipnir-1 not set → trigger() fires
   (modal + howl). If already found → no-op (silent, as it should be)
  - cursor-default maintained even on hover — the tooltip is the only tell

  Why click-to-trigger (not hover-to-trigger)

  Browser autoplay policy: new Audio().play() requires a direct user gesture.
  Click is the only reliable path. Same pattern as the beard button.
