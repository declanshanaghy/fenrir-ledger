## Phase 2 — AuthContext Migration

**Goal:** Replace the current `AuthContext` (which reads `fenrir:auth` from localStorage) with a Clerk-backed context. Anonymous users continue to work identically. Signed-in users get their `householdId` from `user.id`.

### Task 2.1 — Refactor AuthContext to use Clerk hooks

**File(s):** `development/ledger/src/contexts/AuthContext.tsx`
**Depends on:** Tasks 1.3, 1.4

The current `AuthContext` reads `fenrir:auth` from localStorage and manages a `FenrirSession`. After this task, it delegates to Clerk's `useUser()` hook.

```typescript
// development/ledger/src/contexts/AuthContext.tsx
"use client";

import { createContext, useContext, useEffect, useState } from "react";
import { useUser } from "@clerk/nextjs";
import { getOrCreateAnonHouseholdId } from "@/lib/auth/household";
import { isMergeComplete, mergeAnonymousCards } from "@/lib/merge-anonymous";

export type AuthStatus = "loading" | "authenticated" | "anonymous";

export interface AuthContextValue {
  status: AuthStatus;
  householdId: string;
  /** Clerk user.id when signed in, null when anonymous */
  userId: string | null;
  /** Display name from Clerk profile, null when anonymous */
  displayName: string | null;
  /** Avatar URL from Clerk profile, null when anonymous */
  avatarUrl: string | null;
}

const AuthContext = createContext<AuthContextValue | null>(null);

export function AuthProvider({ children }: { children: React.ReactNode }) {
  const { isLoaded, isSignedIn, user } = useUser();
  const [anonHouseholdId] = useState(() => {
    if (typeof window === "undefined") return "";
    return getOrCreateAnonHouseholdId();
  });

  // Run silent anonymous merge when user signs in
  useEffect(() => {
    if (!isLoaded || !isSignedIn || !user) return;
    if (!anonHouseholdId) return;
    if (isMergeComplete(anonHouseholdId)) return;

    mergeAnonymousCards(user.id, anonHouseholdId);
  }, [isLoaded, isSignedIn, user, anonHouseholdId]);

  let status: AuthStatus = "loading";
  let householdId = anonHouseholdId;
  let userId: string | null = null;
  let displayName: string | null = null;
  let avatarUrl: string | null = null;

  if (isLoaded) {
    if (isSignedIn && user) {
      status = "authenticated";
      householdId = user.id;          // Clerk user ID is the householdId
      userId = user.id;
      displayName = user.fullName ?? user.username ?? null;
      avatarUrl = user.imageUrl ?? null;
    } else {
      status = "anonymous";
      // householdId remains anonHouseholdId
    }
  }

  return (
    <AuthContext.Provider value={{ status, householdId, userId, displayName, avatarUrl }}>
      {children}
    </AuthContext.Provider>
  );
}

export function useAuthContext(): AuthContextValue {
  const ctx = useContext(AuthContext);
  if (!ctx) throw new Error("useAuthContext must be used within AuthProvider");
  return ctx;
}
```

**Edge Cases:**
- SSR: `useUser()` returns `{ isLoaded: false }` on the server. `status === "loading"` prevents any storage reads until the client hydrates.
- `anonHouseholdId` is read once on mount. The `useState` initializer guards against SSR.
- Merge runs once per anonymous session (tombstone in localStorage prevents re-merge).

**Definition of Done:** All pages render. `householdId` flows correctly to `getCards()`, `saveCard()`, etc. Anonymous mode unchanged. Signed-in state reads from Clerk.

### Task 2.2 — Update useAuth custom hook

**File(s):** `development/ledger/src/hooks/useAuth.ts`
**Depends on:** Task 2.1

The custom `useAuth` hook currently reads from `AuthContext`. Its public API is preserved; implementation delegates to the refactored context.

```typescript
// development/ledger/src/hooks/useAuth.ts
import { useAuthContext } from "@/contexts/AuthContext";
import type { AuthStatus } from "@/contexts/AuthContext";

export type { AuthStatus };

export interface UseAuthResult {
  status: AuthStatus;
  householdId: string;
  userId: string | null;
  displayName: string | null;
  avatarUrl: string | null;
  isLoading: boolean;
  isAuthenticated: boolean;
}

export function useAuth(): UseAuthResult {
  const { status, householdId, userId, displayName, avatarUrl } = useAuthContext();

  return {
    status,
    householdId,
    userId,
    displayName,
    avatarUrl,
    isLoading: status === "loading",
    isAuthenticated: status === "authenticated",
  };
}
```

**Definition of Done:** All pages that call `useAuth()` compile and run without changes.

## Playwright Testing Strategy with Clerk Testing Tokens

### Overview

Clerk provides `@clerk/testing` which integrates with Playwright to bypass the OAuth browser flow entirely in CI. Tests authenticate programmatically using Testing Tokens issued by Clerk's Backend API.

### Setup

**Global setup file:** `development/ledger/playwright.config.ts` (or `e2e/global-setup.ts`)

```typescript
// e2e/global-setup.ts
import { clerkSetup } from "@clerk/testing/playwright";
import { test as setup } from "@playwright/test";

setup.describe.configure({ mode: "serial" });

setup("global setup", async () => {
  await clerkSetup();
  // clerkSetup() fetches a Testing Token from Clerk Backend API
  // and stores it for the test suite. Requires CLERK_SECRET_KEY env var.
});
```

**Per-test authentication:**

```typescript
// e2e/auth.spec.ts
import { setupClerkTestingToken } from "@clerk/testing/playwright";
import { test, expect } from "@playwright/test";

test("signed-in user sees their cards", async ({ page }) => {
  await setupClerkTestingToken({ page });
  // setupClerkTestingToken injects the token as a query param
  // Clerk bypasses bot detection and returns a valid session
  await page.goto("/");
  // ... assertions
});

test("anonymous user can add a card without signing in", async ({ page }) => {
  // No setupClerkTestingToken — test runs fully anonymous
  await page.goto("/");
  // ... assertions
});
```

### Environment variables for Playwright

```
# Required for E2E tests against Clerk
CLERK_PUBLISHABLE_KEY=pk_test_...    # Same as NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY
CLERK_SECRET_KEY=sk_test_...         # Server-side key — GitHub Actions secret
```

The `CLERK_SECRET_KEY` value must travel only via GitHub Actions secrets (`${{ secrets.CLERK_SECRET_KEY }}`). It must never appear in:
- PR comments
- Commit messages
- Log output
- Test report artifacts

Playwright config passes it via `process.env.CLERK_SECRET_KEY` — GitHub automatically masks it in workflow logs.

### Test user strategy

For testing signed-in flows, create a dedicated test user in the Clerk Development instance:
- Email: `loki-tester@fenrir-ledger.dev` (or similar)
- Provider: GitHub (or email-based in dev mode)
- Created once via Clerk Dashboard → Users → Create user

Testing Tokens let tests authenticate as this user without a real browser OAuth flow. The user's `userId` becomes the `householdId` in test storage assertions.

### CI workflow integration

```yaml
# .github/workflows/playwright.yml (additions)
- name: Run Playwright tests
  env:
    NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY: ${{ secrets.CLERK_PUBLISHABLE_KEY }}
    CLERK_SECRET_KEY: ${{ secrets.CLERK_SECRET_KEY }}
  run: npx playwright test
```

No Testing Token values appear in the workflow YAML — `clerkSetup()` fetches them at runtime using the secret key.

## Definition of Done (Full Integration)

1. `npm run build` succeeds with Clerk SDK installed.
2. Anonymous user flow: open app, add card, data persists — no sign-in prompt, no redirect.
3. Sign-in flow: GitHub OAuth completes, Clerk session established, `householdId` = `user.id`, anonymous cards silently merged.
4. Sign-out flow: Clerk session cleared, app returns to anonymous state with a fresh UUID.
5. Playwright tests pass in CI without any real OAuth browser flow.
6. `CLERK_SECRET_KEY` is never printed in logs, PR comments, or artifacts.
7. No routes are protected — all routes respond to anonymous HTTP requests.
8. `merge-anonymous.ts` prevents double-merge on subsequent sign-ins (tombstone).
